## Goal

Prove the full M1 standalone-cloud core: declarative resources, Discovery, Network, Infrastructure, Autoscaler, and the enhanced scheduler together recover workloads after node loss and keep replicas spread across nodes/zones using one portable manifest.

## Scope

* `demos/25-ha-placement`.
* Three or more runtime nodes from the Docker provider.
* Application with three replicas, anti-affinity, topology spread, requests/limits, and disruption budget.
* Node failure and replacement.
* Optional GPU/stateful placement assertions using fake capacity.

## Demo flow

```text
Apply portable Application manifest
-> Infrastructure creates runtime nodes
-> Scheduler spreads replicas across distinct nodes
-> Discovery publishes Ready endpoints
-> Gateway routes to all healthy replicas
-> stop one runtime node
-> endpoint becomes Unready
-> scheduler replaces lost replica on another eligible node
-> autoscaler creates capacity if needed
-> disruption budget remains satisfied
```
