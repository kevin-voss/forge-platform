## Goal

Give nodes identity beyond capacity: labels (propagated from `NodePool` or set directly on the Runtime agent), taints that repel placements unless tolerated, and automatic architecture/OS facts. Give workloads `placement.nodeSelector` and `placement.tolerations` so a mixed Docker/bare-metal/Hetzner/AWS/Azure, mixed-amd64/arm64 fleet schedules correctly without any code path branching on provider identity.

## Why this step exists

Epic 08's fleet is anonymous beyond node id and capacity — every online node with enough free capacity is a valid candidate. Real fleets are not uniform: some nodes are GPU-only, some are a different CPU architecture, some are reserved for a tenant or workload class, some are mid-drain. This step gives the scheduler the vocabulary to express those constraints (labels + selectors for "must have," taints + tolerations for "must not, unless explicitly allowed") so `25.03`'s affinity/topology work and `25.04`'s priority work both have a stable node-identity model to build on.

## Dependencies

* Step `25.01` — real capacity accounting and the `PlacementTrace` schema this step appends filters to
* Epic `08-multi-node-scheduler`, step `08.02` — node fleet model (`nodes` table, registration)
* Soft dependency: epic `23-forge-infrastructure` — `NodePool` is the intended long-term source of propagated labels/topology; this step works standalone via Runtime agent env in the meantime

## Scope

* `nodes.labels_json`: arbitrary operator-set key/value labels, populated from (a) `NodePool` propagation when epic 23 is present, (b) `FORGE_NODE_LABELS` on the Runtime agent, merged with node-level values winning conflicts
* Automatic reserved labels every node gets for free: `forge.dev/node-id`, `forge.dev/arch`, `forge.dev/os`, `forge.dev/provider`
* `nodes.architecture` (`amd64`|`arm64`) and `nodes.os` (`linux`) columns, populated from Runtime host introspection (`uname`), not operator-declared
* `nodes.taints_json`: array of `{key, value, effect}`, `effect ∈ {NoSchedule, NoExecute}`
* `PlacementRequest.placement.nodeSelector`: equality-match label requirements
* `PlacementRequest.placement.tolerations`: array of `{key, operator: Equal|Exists, value, effect}` matched against candidate node taints
* `spec.platform.{architecture,os}` on the workload (optional; default `amd64`/`linux` — matches every existing demo image, zero behavior change when omitted)
* `NoExecute` taint applied to an already-placed, non-tolerating replica triggers eviction (reuses the `08.05` release-and-reschedule path, not a new mechanism)

## Explicitly out of scope

* Workload-to-workload affinity/anti-affinity and topology spread scoring (`25.03`)
* Priority-aware taint interactions (e.g. "high priority tolerates everything") — taints/tolerations are priority-agnostic in this step
* A soft `PreferNoSchedule`-style taint effect (open question, not built)
* Automatic `NodePool` propagation itself (epic 23's job); this step only defines the merge rule and the standalone env fallback

## Architecture

```text
Node registration (extends 08.02):
  Runtime reports: node_id, address, capacity, labels(from FORGE_NODE_LABELS), taints(from FORGE_NODE_TAINTS),
                    architecture(uname -m), os(uname -s)
  Control merges: reserved labels (forge.dev/*) + pool labels (if NodePool present) + agent-reported labels
                  (node-level wins on key conflict)

Scheduler.place(req) [extends 25.01 trace]:
  candidates = capacityFilter(fleet.online())
  candidates = candidates.filter(matchesNodeSelector(node.labels, req.placement.nodeSelector))
  trace.filters += { name: "node_selector", eliminated: [...] }
  candidates = candidates.filter(matchesPlatform(node.architecture/os, req.platform))
  trace.filters += { name: "platform", eliminated: [...] }
  candidates = candidates.filter(node.taints.all { toleratedBy(req.placement.tolerations) })
  trace.filters += { name: "taints", eliminated: [...] }
  # 25.03/25.04 filters compose after this; 08.03 strategy picks among final candidates

NoExecute added to a node with existing non-tolerating placements:
  → those placements marked lost, capacity released, fresh placement requested (08.05 path, unchanged)
```

## Components

| Component | Responsibility |
|---|---|
| `NodeLabelMerger` | Combine reserved + pool + agent-reported labels on registration |
| `NodeSelectorFilter` | Equality-match `placement.nodeSelector` against `node.labels` |
| `PlatformFilter` | Match `spec.platform.{architecture,os}` against node facts |
| `TaintTolerationFilter` | Eliminate nodes whose taints aren't tolerated by the request |
| `TaintChangeHandler` | On `NoExecute` taint added, evict non-tolerating placements via the `08.05` reschedule path |

## Contracts

* `POST /v1/nodes/register` (extends `08.02`):

```json
{ "node_id": "node-a", "address": "http://runtime-a:4102",
  "capacity": { "cpu_millis": 4000, "mem_mb": 4096 },
  "labels": { "disk": "ssd", "tier": "standard" },
  "taints": [ { "key": "dedicated", "value": "db", "effect": "NoSchedule" } ],
  "architecture": "arm64", "os": "linux" }
```

All new fields optional; an unmodified epic-08 Runtime agent registers exactly as before with empty labels/taints and default `architecture=amd64`/`os=linux`.

* `POST /v1/placements` request gains:

```json
{ "placement": {
    "nodeSelector": { "disk": "ssd" },
    "tolerations": [ { "key": "dedicated", "operator": "Equal", "value": "db", "effect": "NoSchedule" } ]
  },
  "platform": { "architecture": "arm64" } }
```

* `GET /v1/nodes` response gains `labels`, `taints`, `architecture`, `os`

## Data model

```sql
ALTER TABLE nodes
  ADD COLUMN labels_json  JSONB NOT NULL DEFAULT '{}',
  ADD COLUMN taints_json  JSONB NOT NULL DEFAULT '[]',
  ADD COLUMN architecture TEXT  NOT NULL DEFAULT 'amd64',
  ADD COLUMN os           TEXT  NOT NULL DEFAULT 'linux';

ALTER TABLE placements
  ADD COLUMN node_selector_json JSONB,
  ADD COLUMN tolerations_json   JSONB NOT NULL DEFAULT '[]',
  ADD COLUMN platform_json      JSONB;
```

Every existing node row defaults to `labels={}`, `taints=[]`, `architecture=amd64`, `os=linux` — identical candidacy to epic 08 until an operator sets a label/taint.

## Configuration

```text
FORGE_NODE_LABELS          # k1=v1,k2=v2 on the Runtime agent (merged; node-level wins)
FORGE_NODE_TAINTS          # key=value:Effect,... e.g. dedicated=db:NoSchedule
FORGE_NODE_POOL_ID         # NodePool identifier this node belongs to (epic 23); optional
FORGE_STRICT_NODE_SELECTOR # default false; when true, an empty nodeSelector match set is a hard error not silent no-op
```

## Security considerations

* Labels/taints carry operator intent, never secrets; safe to log and return via the fleet API
* A tenant workload cannot set another tenant's node's labels/taints — only the Runtime agent (at registration) or `NodePool` (epic 23, platform-operator-owned) can
* `NoSchedule`/`NoExecute` are the only effects; there is no way for a toleration to grant access to capacity a project's isolation boundary would otherwise deny (isolation is enforced above scheduling, per the cross-cutting requirements)

## Observability

* Log node registration with resolved (merged) labels/taints; log every `taint_added`/`taint_removed` transition
* Metrics: `forge_nodes_total{architecture,os}`, `forge_placement_filtered_total{filter="node_selector"|"platform"|"taints"}`, `forge_taint_evictions_total`
* Trace span `scheduler.place` gains `filters_applied` attribute listing filter names in order

## Failure handling

* `nodeSelector` matches zero nodes → placement goes `pending` (`08.04` queue) with reason `NoNodeMatchesSelector`, not an error
* Conflicting labels between pool and node → node-level value wins, logged as a conflict (visible, not silent)
* `NoExecute` taint added to a node with placements that have no toleration → eviction proceeds even under load; a flapping taint (`added` then quickly `removed`) does not un-evict already-rescheduled replicas (same fencing discipline as `08.05`)
* Runtime reports an unrecognized architecture string → stored as-is; `PlatformFilter` treats it as simply not matching `amd64`/`arm64` requests (never crashes)

## Unit tests

* `NodeLabelMerger`: pool label + node label with same key → node label wins; distinct keys → both present
* `NodeSelectorFilter`: node missing a required label key is eliminated; exact match passes
* `PlatformFilter`: `arm64` request eliminates `amd64` nodes and vice versa; omitted `platform` matches any node
* `TaintTolerationFilter`: `NoSchedule` taint without matching toleration eliminates the node; `Exists` operator matches any value for the key
* `TaintChangeHandler`: adding `NoExecute` to a node evicts only that node's non-tolerating placements

## Integration tests

* Register two nodes, one `arm64` one `amd64`; deploy with `platform.architecture=arm64` → placed only on the `arm64` node
* Taint a node `NoSchedule`; deploy without a matching toleration → never placed there; deploy with the toleration → placed there
* Apply `NoExecute` taint to a node running placements → those placements rescheduled elsewhere (or pending) within one scheduler tick
* Legacy deploy with no `placement`/`platform` fields on an unmodified fleet → identical outcome to epic 08
