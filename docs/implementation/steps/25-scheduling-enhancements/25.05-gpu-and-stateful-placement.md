## Goal

Extend placement for GPU workloads, capacity reservations, and stateful services that require stable node/volume affinity. This prepares the scheduler for AI workloads, databases, volumes, and safe node scale-down.

## Dependencies

* `25.01` - resource requests and capacity accounting.
* `25.02` - node labels and selectors.
* `25.04` - disruption budgets and preemption.
* Epic `30-forge-volumes` for long-term volume semantics; this step defines the scheduler contract first.

## Scope

* GPU resource accounting by count, vendor, model, memory, and driver capability.
* Placement reservations with TTL and owner reference.
* Stateful placement constraints: stable node affinity, volume locality, primary/replica anti-affinity.
* Scheduler refusal to preempt or drain protected primaries unless explicitly approved.
* `Node` status includes allocatable GPU and reservation fields.

## Explicitly out of scope

* Model serving runtime and batching.
* Distributed block storage implementation.
* GPU node creation; node autoscaler requests capacity through NodePools.

## Contracts

```yaml
placement:
  resources:
    gpu:
      count: 1
      vendor: nvidia
      memory: 24Gi
  stateful:
    volumeRef: invoice-db-data
    role: primary
    migrationPolicy: manual-approval
```

Reservation:

```yaml
apiVersion: forge.dev/v1
kind: Reservation
metadata: { name: model-warm-pool-1 }
spec:
  resources: { cpu: "4", memory: 16Gi, gpu: { count: 1 } }
  expiresAfter: 30m
```

## Observability

* Metric `forge_scheduler_gpu_allocatable{node,model}`.
* Metric `forge_scheduler_reservations_active_total`.
* Log stateful placement decisions with `{deployment, volume, selected_node, reason}`.

## Failure handling

* No matching GPU -> pending reason `InsufficientGpu`.
* Reservation expires -> capacity is released and event emitted.
* Stateful primary drain requested -> blocked unless an explicit migration approval exists.

* Reservations hold capacity until used or expired.
* Stateful workloads prefer and preserve volume locality.
* Protected primaries are not preempted or drained automatically.
