## Goal

Deliver `demos/14-model-serving`: a Go client that sends text to `forge-models` and receives an embedding, a classification, and a generated summary from a local backend — plus a one-command acceptance suite that is the epic gate.

## Why this step exists

The epic is done only when a real client proves all three capabilities end-to-end against a running service using a local model, with usage observable. This assembles prior slices into a reproducible demo for CI/regression.
