## Goal

Add `POST /v1/models/{model}/generate`, `/classify`, and `/summarize` served by a local/fake generation adapter, giving the platform the three text capabilities agents and workflows depend on, with deterministic CI behavior.

## Why this step exists

Agents (15) call generate/classify/summarize for diagnosis and summaries. This step completes the synchronous inference surface over the stable API while keeping CI deterministic via a fake adapter.

## Dependencies

* `14.03` (embeddings + adapter path proven)

## Scope

* `LocalGenerationAdapter` (deterministic fake for CI; optional real small model)
* `POST /v1/models/{model}/generate` — prompt → text + finish reason
* `POST /v1/models/{model}/classify` — text + labels → scored labels
* `POST /v1/models/{model}/summarize` — text → summary (thin wrapper over generate)
* Request validation + capability checks
* Token/usage counts in responses (approximate for fake)

## Explicitly out of scope

* Streaming + async job mode (14.05)
* Usage metrics export/CLI (14.06)
* Real vendor adapters with credentials

## Architecture

```text
/generate  → adapter.generate(prompt, params) → {text, finish_reason, usage}
/classify  → adapter.classify(text, labels)   → {labels:[{label,score}]}
/summarize → adapter.generate(summarize_prompt(text)) → {summary, usage}
```

Fake adapter is deterministic: generation echoes/transforms input via a seeded rule; classification scores labels by a stable similarity to the input (reusing the embedding adapter). This yields reproducible, assertable outputs.

## Components

| Component | Purpose |
|---|---|
| `LocalGenerationAdapter` | Deterministic generate/classify |
| `generate`/`classify`/`summarize` routes | Validate + dispatch |
| `summarize_prompt` | Build a summarization prompt for generate |

## Contracts

* `POST .../generate` `{ "prompt":"...", "max_tokens":128, "temperature":0 }` → `200 { "text","finish_reason","usage" }`
* `POST .../classify` `{ "input":"...", "labels":["a","b"] }` → `200 { "labels":[{"label","score"}] }` (sorted desc)
* `POST .../summarize` `{ "input":"..." }` → `200 { "summary","usage" }`
* `422` on capability mismatch/invalid params; `404` unknown model
* Extend OpenAPI

## Data model

No persistence. `usage = { prompt_tokens, completion_tokens, total_tokens }` (approximate counts for fake).
