## Goal

Make model usage observable (per-model request counts, tokens, latency exported to the Observe stack), finalize and validate the complete OpenAPI contract, and optionally add a thin `forge model` CLI subcommand.

## Why this step exists

`specs.md` Step 14 requires that model usage is observable and that adapters use one stable, documented API. This step hardens the contract and telemetry before the epic gate.

* `14.05` (all inference modes present)

## Architecture

```text

```

## Components

| Component | Purpose |
|---|---|
| Metrics middleware | Instrument every inference route |
| `/metrics` exporter | Prometheus endpoint (Observe scrapes) |
| `/v1/usage` | Human-readable aggregate |
| `forge model` CLI (opt) | Thin client for list/embed/generate |

## Contracts

* `GET /metrics` â†’ Prometheus text format
* `GET /v1/usage` â†’ `200 { "by_model": { "<id>": {"requests","tokens","errors","p95_latency_ms"} } }`
* Full OpenAPI covers: models, embed, generate, classify, summarize, jobs, usage
* CLI (optional): `forge model list`, `forge model embed --model <m> --text <t>`

## Data model

In-memory counters/histograms; snapshot for `/v1/usage`. No persistence.

## Configuration

```text

FORGE_OTEL_EXPORTER_OTLP_ENDPOINT  # optional OTEL push target
```

## Security considerations

* `/metrics` internal-only (not routed publicly)
* Large prompt/response bodies not logged in full (truncate/redact)

## Observability

* This step *is* the observability step: counters, histograms, error rates per model
* Verify metrics appear via Observe (12) scrape in integration

## Failure handling

* Metrics recording never blocks/breaks an inference request (best-effort)
* Exporter failure logged, not fatal
