# Step 14.02: Model registry + `GET /v1/models`

## Status

Not started

## Epic

`14-forge-models` → [`../../epics/14-forge-models.md`](../../epics/14-forge-models.md)

## Goal

Introduce an adapter interface and a model registry that enumerates available models with their capabilities (embed/generate/classify/summarize), backend, and embedding dimension, exposed via `GET /v1/models` and `GET /v1/models/{model}`.

## Why this step exists

Every inference endpoint dispatches through a registry so the platform never hardcodes a vendor. Publishing capabilities and embedding dimension is also required by Memory (17), which must know the vector size.

## Dependencies

* `14.01` (service skeleton)

## Scope

* `ModelAdapter` abstract interface (capabilities, metadata, health)
* Registry loading from config/definitions (`models.yaml`)
* `GET /v1/models`, `GET /v1/models/{model}` (registry read)
* Per-model `GET /v1/models/{model}/health`
* Register a placeholder/`fake` adapter so the registry is non-empty (real embed adapter in 14.03)

## Explicitly out of scope

* Actual embedding/generation computation (14.03/14.04)
* Streaming/async (14.05)
* Usage metrics (14.06)

## Architecture

```text
models.yaml → RegistryLoader → { model_id → ModelAdapter }
GET /v1/models → serialize registry entries (capabilities, dim, backend, status)
```

`ModelAdapter` declares `capabilities: set[{embed,generate,classify,summarize}]`, `embedding_dim: int|None`, `backend: str`, and `health() -> ok|degraded|down`.

## Components

| Component | Purpose |
|---|---|
| `ModelAdapter` | Interface all backends implement |
| `RegistryLoader` | Parse `models.yaml`, instantiate adapters |
| `registry` routes | List/get models + per-model health |
| `FakeAdapter` | Deterministic placeholder to populate registry |

## Contracts

* `GET /v1/models` → `200 { "models":[{ "id","capabilities","backend","embedding_dim","status" }] }`
* `GET /v1/models/{model}` → `200 {model}` | `404 model_not_found`
* `GET /v1/models/{model}/health` → `200 { "status":"ok|degraded|down" }`
* Extend OpenAPI

## Data model

Registry entry (in-memory, sourced from `models.yaml`):

```yaml
models:
  - id: local-embed-small
    backend: local
    capabilities: [embed]
    embedding_dim: 384
  - id: local-general
    backend: fake
    capabilities: [generate, classify, summarize]
```

## Configuration

```text
FORGE_MODELS_CONFIG   # path to models.yaml (default app/models.yaml)
FORGE_MODELS_BACKEND  # selects default backend family (fake|local)
```

## Security considerations

* Registry is read-only over HTTP (no runtime model registration via API in this step)
* No secret material in `models.yaml`

## Observability

* Log registry load (count, ids, backends)
* Metric: `models_registry_size`, per-model health gauge

## Failure handling

* Malformed `models.yaml` → startup failure with clear error
* Unknown model id → `404`
* Adapter health down → reported in list, not a service-wide failure
