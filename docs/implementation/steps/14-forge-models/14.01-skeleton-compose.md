# Step 14.01: Skeleton + Compose

## Status

Not started

## Epic

`14-forge-models` → [`../../epics/14-forge-models.md`](../../epics/14-forge-models.md)

## Goal

Stand up the `forge-models` Python (FastAPI) service with health endpoints, structured logging, graceful shutdown, a Docker image, and Compose wiring on host port `4300` — no inference logic yet.

## Why this step exists

All model capabilities need a running, contract-compliant Python service and a place for adapters to plug in. Establishing the skeleton first isolates ML/runtime concerns from the HTTP/ops scaffold.

## Dependencies

* Epic `00-repository-foundation` complete
* Epic `01-runtime-contract` conventions (health, logs, `PORT`, SIGTERM)

## Scope

* FastAPI app under `services/forge-models`
* `GET /health/live`, `GET /health/ready`, `GET /` identity JSON
* Config via env (`PORT`, `FORGE_LOG_LEVEL`, `FORGE_MODELS_BACKEND`)
* Structured JSON logging + request IDs
* Dockerfile (slim Python base) + Compose service on `4300`
* Makefile targets; port reservation confirmed in `docs/operations/ports.md`

## Explicitly out of scope

* Model registry (14.02)
* Embeddings (14.03)
* Generate/classify/summarize (14.04)
* Streaming/async (14.05)
* Metrics/CLI (14.06)

## Architecture

```text
services/forge-models/
├── app/
│   ├── main.py         # FastAPI app + lifespan (startup/shutdown)
│   ├── config.py       # env settings (pydantic-settings)
│   ├── health.py       # live/ready routes
│   └── logging.py      # JSON log config + request id middleware
├── pyproject.toml
├── Dockerfile
└── README.md
```

Readiness returns 200 only after backend selection is validated (backend impl arrives in 14.02/14.03; here it just confirms config parses).

## Contracts

* `GET /health/live` → `200 {"status":"live"}`
* `GET /health/ready` → `200 {"status":"ready"}`
* `GET /` → `200 {"service":"forge-models","language":"python","status":"running","version":"<v>"}`
* Create `contracts/openapi/forge-models.openapi.yaml` (3 paths; FastAPI can also emit `/openapi.json`, but keep the canonical file in `contracts/`)

## Configuration

```text
PORT                    # Compose maps host 4300
FORGE_LOG_LEVEL         # debug|info|warn|error
FORGE_MODELS_BACKEND    # fake|local (default fake); validated at startup
FORGE_SERVICE_NAME      # default forge-models
FORGE_SERVICE_VERSION
```

## Security considerations

* Internal service; not publicly exposed by Gateway
* No secrets required for `fake`/`local` backends
* No secrets logged

## Observability

* JSON logs with `timestamp`, `level`, `service`, `message`, `request_id`
* OTEL recommended, not required this step

## Failure handling

* Unknown backend value → startup failure (exit non-zero)
* SIGTERM → drain, exit 0 within grace period
* Invalid `PORT` → exit non-zero
