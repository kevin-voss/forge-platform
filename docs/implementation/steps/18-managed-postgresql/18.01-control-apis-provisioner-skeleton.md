## Goal

Introduce the managed-database resource model and read/write APIs in Control (instances, databases, credentials, attachments, backups) plus a `Provisioner` interface with a no-op/fake implementation — establishing the management-plane surface before any real Postgres is provisioned.

## Why this step exists

The management plane needs a stable resource model and API contract before wiring real provisioning. A `Provisioner` seam lets later steps swap the fake for a real Go provisioner without changing the API, and keeps CI deterministic.

## Dependencies

* Epic `02-forge-control` resource model + API conventions (`02.04`/`02.05`)
* Epics `00`, `01` conventions

## Scope

* Control migrations + domain model: `db_instance`, `db_database`, `db_credential`, `db_attachment`, `db_backup`
* CRUD-read APIs: create instance (record only), list/get instances/databases
* `Provisioner` interface (create/delete instance, create db/role, backup/restore, rotate) + `FakeProvisioner`
* Resource state machine (`provisioning|available|error|deleting`)
* Explicit isolation invariant: managed DB records reference product infra, never Control's own DB connection

## Explicitly out of scope

* Real Postgres container provisioning (18.02)
* Attach + injection (18.03)
* Backup/restore execution (18.04)
* Rotation/deletion protection (18.05)
* CLI/demo (18.06)

## Architecture

```text
Control API → ManagedDbService → Provisioner (Fake now; real Go later)
DB tables: db_instance, db_database, db_credential, db_attachment, db_backup
state: provisioning → available | error; delete → deleting → gone
```

## Components

| Component | Purpose |

## Data model

```sql
CREATE TABLE db_instance (
  id UUID PRIMARY KEY, project_id TEXT NOT NULL, name TEXT NOT NULL,
  status TEXT NOT NULL, engine TEXT NOT NULL DEFAULT 'postgres',
  deletion_protection BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ, UNIQUE(project_id, name)
);
CREATE TABLE db_database ( id UUID PRIMARY KEY, instance_id UUID REFERENCES db_instance(id), name TEXT NOT NULL, created_at TIMESTAMPTZ );
CREATE TABLE db_credential ( id UUID PRIMARY KEY, database_id UUID REFERENCES db_database(id), username TEXT NOT NULL, secret_ref TEXT, status TEXT NOT NULL, created_at TIMESTAMPTZ );
CREATE TABLE db_attachment ( id UUID PRIMARY KEY, database_id UUID REFERENCES db_database(id), application_id UUID NOT NULL, env_var TEXT NOT NULL DEFAULT 'DATABASE_URL', created_at TIMESTAMPTZ );
CREATE TABLE db_backup ( id UUID PRIMARY KEY, database_id UUID REFERENCES db_database(id), location TEXT, status TEXT NOT NULL, created_at TIMESTAMPTZ );
```

## Configuration

## Security considerations

* Managed-db records are project-scoped
* Isolation invariant enforced: no managed database uses Control's own DB credentials/connection
* Credentials stored as `secret_ref` (pointer to Secrets), never plaintext in these tables

## Observability

* Log resource lifecycle transitions
* Metric: `managed_db_instances_total{status}`

## Failure handling

* Duplicate instance name (per project) → `409`
* Provisioner error → instance `error` with reason
* Missing resource → `404`
