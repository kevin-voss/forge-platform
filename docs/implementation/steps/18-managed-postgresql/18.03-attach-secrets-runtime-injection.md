## Goal

Attach a managed database to an application so that, on deploy, Runtime injects a connection URL (e.g. `DATABASE_URL`) sourced from Secrets — the product receives working database access with no hardcoded credentials.

## Why this step exists

`specs.md` Step 18 requires attaching to an application and injecting the connection URL; acceptance requires products receive no hardcoded credentials. This connects managed DB to Secrets (10) and Runtime (04).

## Dependencies

* `18.02` (databases + credentials in Secrets)
* Epic `10-forge-secrets` runtime delivery (`10.04`)
* Epic `04-forge-runtime` env injection (`04.03`)

## Scope

* `POST /v1/databases/{database_id}/attach` `{ application_id, env_var }`
* Compose the connection URL from host/port/db + credential secret → store as a project secret
* On deploy, Runtime injects the URL env var from Secrets into the app workload
* Detach endpoint; multiple attachments (different apps) supported
* Ensure URL is masked in logs (via Secrets masking conventions)

## Explicitly out of scope

* Backup/restore (18.04)
* Rotation (18.05 — attach must survive rotation later)
* CLI/demo (18.06)

## Architecture

```text
attach → build DATABASE_URL from {user,pass(secret),host,port,db} → store secret (10)
       → record db_attachment(application, env_var)
deploy → Runtime pulls attachment's secret → injects env_var into workload (04.03)
```

## Components

| Component | Purpose |
|---|---|
| Attach handler | Bind database ↔ application |
| URL composer | Build connection URL from parts |
| Secrets integration | Store/deliver the URL secret |
| Runtime injection hook | Inject env var at deploy |

## Contracts

* `POST /v1/databases/{database_id}/attach` → `201 { attachment }`
* `DELETE /v1/databases/attachments/{id}` → `204`
* `GET /v1/applications/{app_id}/databases` → attachments for an app
* Injected env var default `DATABASE_URL` (configurable per attachment)
* Extend OpenAPI

## Data model

Uses `db_attachment` (18.01). The composed URL is stored as a Secrets entry referenced by the attachment; only the reference is in Control.

## Configuration

```text
FORGE_DB_DEFAULT_ENV_VAR    # default DATABASE_URL
FORGE_SECRETS_URL
FORGE_RUNTIME_URL
```

## Security considerations

* Connection URL stored only in Secrets; Control keeps a reference
* URL masked in all logs (Secrets masking, epic 10)
* Attach authz: caller must have write on both the database and the application (project-scoped)
* Detach revokes injection on next deploy

## Observability

* Log attach/detach (app, db, env var) — URL never logged
* Metric: `managed_db_attachments_total`

## Failure handling

* Attach to app in another project → `403`/`404`
* Missing database/app → `404`
* Runtime injection failure at deploy → deploy fails with clear cause (no silent missing URL)
