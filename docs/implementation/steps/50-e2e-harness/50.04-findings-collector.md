## Goal

Implement the only writer to [`PLATFORM_FINDINGS.md`](../../../demo-projects/PLATFORM_FINDINGS.md)
and `artifacts/findings.json`, plus the `platform.expect(service, fn)` assertion wrapper that routes
platform-behaviour failures into findings instead of hard test failures.

## Dependencies

* `50.03`; the findings template + doc under `docs/demo-projects/`.

## Scope

* `harness/findings.ts`: `record({service, demo, severity, title, tested, expected, actual,
  evidence, repro})`; append a markdown block from `findings-template.md`; update summary/by-service/
  by-demo counters; write/merge `findings.json`; dedupe by `service+title`.
* `platform.expect(service, fn, {severity})`: on throw → `findings.record(...)` with captured
  evidence (status, body, logs, traceId, screenshot); `blocker` marks the product failed, `major`/
  `minor` mark it degraded and continue.
* Clear separation from ordinary Playwright `expect` (product assertions stay hard failures).
