## Goal

Define the `demo.json` contract every demo product ships and a lifecycle runner (`demo.ts`) that
takes a product from `up → ready → seed → test → down`, plus a `forge.ts` wrapper around the CLI.

## Dependencies

* `50.01` harness skeleton.

## Scope

* `demo.json` JSON schema: `id, title, epic, compose?, deploy, seed?, hosts[], baseURL, spec,
  services[], teardown` (see [`e2e-harness.md` §2](../../../demo-projects/e2e-harness.md)).
* `harness/demo.ts`: load `demo.json`, run `deploy`/`seed`/`teardown` scripts, expose `baseURL`.
* `harness/forge.ts`: thin wrapper for `forge build`, `forge apply`, `forge get/wait`, `forge logs`.
* Decision (see epic open question): adopt **one shared minimal SPA style** for all products;
  document it as `docs/demo-projects/shared-spa-style.md` or a section here.

## Explicitly out of scope

* Preflight/health (50.03), findings (50.04), orchestration (50.05).

## Implementation tasks

1. Write `demo.schema.json` + a loader/validator.
2. Implement `demo.ts` lifecycle around child-process script execution with timeouts + log capture.
3. Implement `forge.ts` using the installed `forge` CLI; surface non-zero exits clearly.
4. Document the shared SPA convention.
