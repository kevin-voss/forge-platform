## Goal

Stand up `services/forge-network` as a health-checked Go service on host port `4110` that
owns a cluster-wide, provider-independent address plan: a single flat CIDR carved into one
`/24` per node and, within that, one address per workload. Every later step in this epic —
node join, WireGuard peers, network modes, policy enforcement, DNS integration — assigns
addresses out of the model this step establishes.

## Why this step exists

Epics 04–08 never needed a real address plan because every simulated node shared one
Docker bridge. The moment nodes can be on different hosts, providers, or continents, they
need a coordinated, collision-free address space with a service that owns allocation and
survives restarts. Building the plan and its persistence before anything joins keeps every
later step (bootstrap join, peer computation, policy rules) working against stable,
queryable state instead of ad hoc math.

## Scope

* New service `services/forge-network` (Go), port `4110`, `/health/live` + `/health/ready`
* `Network` kind (cluster-scoped, generic envelope): `clusterCidr`, `nodePrefixLength`,
  reserved `ipv6Cidr`
* Node address leases: one `/24` block per node, carved sequentially from the cluster CIDR
* Workload address leases: one address per workload, carved from its node's `/24`
* Lease release + reclamation (explicit release on delete; orphan sweep for leases whose
  owning node/workload disappeared without releasing)
* Collision rules: refuse to activate a cluster CIDR that overlaps an existing Docker
  bridge subnet on the host, or a provider private-network CIDR declared on the install
  target
* OpenAPI for `Network` + lease endpoints

## Explicitly out of scope

* Node identity, bootstrap tokens, join handshake (`22.02`)
* WireGuard key registry and peer distribution (`22.03`)
* Network mode selection / per-pair transport (`22.04`)
* `NetworkPolicy` (`22.05`)
* Discovery/DNS publication (`22.06`)
* IPv6 data plane (field reserved only)

## Data model

```sql
CREATE SCHEMA IF NOT EXISTS network;

CREATE TABLE network.networks (
  id                 TEXT PRIMARY KEY,
  name               TEXT NOT NULL UNIQUE,
  cluster_cidr       CIDR NOT NULL,
  node_prefix_length INT  NOT NULL DEFAULT 24,
  ipv6_cidr          CIDR,
  generation         INT  NOT NULL DEFAULT 1,
  resource_version   BIGINT NOT NULL DEFAULT 1,
  phase              TEXT NOT NULL DEFAULT 'Ready',
  created_at         TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE network.node_leases (
  network_id  TEXT NOT NULL REFERENCES network.networks(id),
  node_id     TEXT NOT NULL,
  cidr        CIDR NOT NULL,
  gateway     INET NOT NULL,
  leased_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  released_at TIMESTAMPTZ,
  PRIMARY KEY (network_id, node_id)
);
CREATE UNIQUE INDEX node_leases_active_cidr
  ON network.node_leases (network_id, cidr) WHERE released_at IS NULL;

CREATE TABLE network.workload_leases (
  network_id  TEXT NOT NULL REFERENCES network.networks(id),
  node_id     TEXT NOT NULL,
  workload_id TEXT NOT NULL,
  address     INET NOT NULL,
  leased_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  released_at TIMESTAMPTZ,
  PRIMARY KEY (network_id, workload_id)
);
CREATE UNIQUE INDEX workload_leases_active_address
  ON network.workload_leases (network_id, address) WHERE released_at IS NULL;
```

## Security considerations

* API is internal (platform trust boundary, same as Control/Runtime); no secrets in any
  lease payload
* Unique partial indexes prevent two active leases claiming the same block or address
* Startup collision check refuses to activate a cluster CIDR that overlaps an existing
  Docker bridge subnet (queried via the Docker API) or a declared provider private-network
  CIDR — logged and surfaced as a `Failed` condition with a clear reason, never silently
  narrowed

## Observability

* Log fields on allocate/release: `{network, node_id, workload_id, cidr|address, action}`
* Metrics: `forge_network_node_leases_total{status=active|released}`,
  `forge_network_workload_leases_total{status=...}`, `forge_network_free_node_blocks`
* Trace spans: `network.lease.allocate`, `network.lease.release`

## Failure handling

* Cluster CIDR exhausted (no free `/24` block) → typed `NoAddressSpaceAvailable`, `409`
* Node's `/24` exhausted (>253 workload addresses) → typed `NodeBlockExhausted`, `409`
  (operator response: shorten the node prefix or reduce workloads-per-node)
* CIDR collision at startup → service reports `Failed` `Network` condition with reason
  `CidrCollision`, refuses to allocate until corrected
* Duplicate lease request for an already-leased node/workload → returns the existing lease
  (idempotent)

## Manual verification

```bash
make dev
curl -s -X POST localhost:4110/v1/networks \
  -d '{"name":"cluster-overlay","spec":{"clusterCidr":"10.100.0.0/16","nodePrefixLength":24}}' \
  -H 'content-type: application/json' | jq '{name: .metadata.name, spec}'

curl -s -X POST localhost:4110/v1/networks/cluster-overlay/node-leases \
  -d '{"node_id":"node-a"}' -H 'content-type: application/json' | jq
# expect cidr "10.100.1.0/24"

curl -s -X POST localhost:4110/v1/networks/cluster-overlay/node-leases \
  -d '{"node_id":"node-b"}' -H 'content-type: application/json' | jq
# expect cidr "10.100.2.0/24" (next free block, no collision with node-a)

curl -s -X POST localhost:4110/v1/networks/cluster-overlay/workload-leases \
  -d '{"node_id":"node-a","workload_id":"wl_1"}' -H 'content-type: application/json' | jq
# expect address inside 10.100.1.0/24
```

## Acceptance criteria

* `forge-network` runs on `4110` with passing `/health/live` and `/health/ready`
* A `Network` can be created with a cluster CIDR and node prefix length and reaches
  `Ready`
* Node leases are unique, sequential, and idempotent per node id
* Workload leases are unique within their node's block and idempotent per workload id
* An overlapping cluster CIDR (vs. an existing Docker bridge subnet) is refused at startup
  with a clear reason
* Releasing a lease makes its block/address available for reuse without colliding with
  still-active leases
