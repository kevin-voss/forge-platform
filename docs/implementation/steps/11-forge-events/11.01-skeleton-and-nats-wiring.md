# Step 11.01: Skeleton + NATS wiring

## Status

Not started

## Epic

`11-forge-events` → [`../../epics/11-forge-events.md`](../../epics/11-forge-events.md)

## Goal

Create the Forge Events service: a Go HTTP service on host port `4105` satisfying the runtime contract (health, structured logs, env config, graceful shutdown), connected to NATS JetStream, that bootstraps the platform streams on startup and reports readiness only when JetStream is reachable and streams exist.

## Why this step exists

Publish/subscribe (11.02) and durable consumers (11.03) require a running service with a healthy JetStream connection and pre-created streams. Building the skeleton + stream bootstrap first isolates infrastructure wiring from the messaging API.

## Dependencies

* Foundation `00` — NATS JetStream running under Compose
* Epic `01-runtime-contract` — health/log/env conventions
* No prior steps in this epic

## Scope

* Go HTTP service (net/http or chi) with `/health/live`, `/health/ready`, identity `/`, structured JSON logs, env config, SIGTERM graceful shutdown
* NATS JetStream client connection with reconnection handling
* Stream bootstrap: idempotently create per-family streams (`build`, `deployment`, `runtime`, `application`, `agent`) with subjects (e.g. `build.>`)
* Readiness gate: ready only when JetStream connected + streams present
* Dockerfile + Compose service `forge-events` (port 4105) depending on NATS
* Skeleton OpenAPI

## Explicitly out of scope

* Publish/subscribe API (11.02)
* Durable consumers/ack/retry (11.03)
* DLQ (11.04), schemas (11.05), idempotency/identity (11.06)

## Architecture

```text
services/forge-events/
├── cmd/forge-events/main.go   # bootstrap, health, shutdown
├── internal/config/config.go
├── internal/nats/conn.go      # connect + reconnect
├── internal/nats/streams.go   # idempotent stream creation
└── internal/health/health.go
```

Streams (JetStream):

```text
build       subjects: build.>
deployment  subjects: deployment.>
runtime     subjects: runtime.>
application subjects: application.>
agent       subjects: agent.>
```

## Contracts

* `GET /health/live` / `GET /health/ready` (ready requires JetStream + streams)
* `GET /` identity `{ "service":"forge-events","language":"go","status":"running" }`
* `contracts/openapi/forge-events.openapi.yaml` — skeleton with the three paths

## Data model

No SQL. State lives in JetStream (streams). No domain persistence yet.


## Security considerations

* No secrets; NATS credentials (if any) via env, never logged
* Health/identity endpoints unauthenticated per contract
* Auth on publish/subscribe deferred to 11.06 (optional)

## Observability

* Log NATS connect/reconnect events + stream bootstrap results
* Metrics: `forge_events_ready` gauge, `forge_nats_reconnects_total`, `forge_streams_total`
* Trace span `events.bootstrap`

## Failure handling

* NATS unreachable at startup → not ready; retries connection with backoff (liveness stays 200)
* Stream creation is idempotent (create-if-absent); a pre-existing compatible stream is accepted
* NATS disconnect at runtime → readiness flips to non-200 until reconnected
* SIGTERM → drain in-flight requests, close NATS cleanly, exit 0
