## Goal

Assemble `demos/11-event-driven`: a Go producer publishes `application.crashed` events through Forge Events to an Elixir consumer that processes them, proving polyglot delivery. The demo also injects a failing message to prove retry → DLQ, and restarts the consumer plus re-publishes a duplicate to prove idempotency (exactly-once logical processing). Wire into `make demo DEMO=11` as the epic acceptance gate.

## Why this step exists

The events pieces (publish/subscribe, durability, DLQ, schemas, idempotency) only prove the epic when a single command demonstrates cross-language delivery, retry/DLQ, and idempotent processing against the real NATS-backed stack. This is the gate in `MASTER_PLAN.md`.

## Acceptance criteria

* `make demo DEMO=11` runs all phases and exits 0
* A Go producer's events are delivered to and processed by an Elixir consumer (polyglot)
* A malformed event is rejected at publish (`422`) and never delivered
* A poison message is retried up to the limit and lands in an inspectable DLQ
* A consumer restart plus a duplicate publish results in exactly-once logical processing (idempotency)
* Producer/consumer satisfy the runtime contract; stack tears down on exit
