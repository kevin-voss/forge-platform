## Goal

Implement streamed object upload and download so large files transfer without being buffered entirely in memory, writing payloads to the content-addressed FS layout and recording size/content-type in metadata.

## Why this step exists

Storage's core value is moving bytes for artifacts, backups, and model files that can be large. Streaming I/O with bounded memory is a hard acceptance criterion in `specs.md` Step 13 and must be established before integrity, ranges, and signing build on it.

## Dependencies

* `13.02` (buckets + object metadata + project isolation)

## Scope

* `PUT /v1/buckets/{bucket}/objects/{key}` — streamed upload to temp file, then atomic move into `objects/`
* `GET /v1/buckets/{bucket}/objects/{key}` — streamed download from FS
* `HEAD` for metadata; object metadata update (size, content-type, updated_at)
* Bounded-memory streaming both directions (fixed-size buffers)
* Overwrite semantics (last write wins for a key)

## Explicitly out of scope

* SHA-256 verification + dedup finalization (13.04)
* Range requests (13.04)
* Signed tokens (13.05)
* Quota enforcement (13.06)

## Architecture

```text
PUT  → stream body → tmp file (meta/tmp/<uuid>) → fsync → rename into objects/<layout> → upsert metadata
GET  → open file → stream in fixed chunks → response body
```

Interim object path (pre-content-addressing): `objects/<project_id>/<bucket_id>/<key-hash>`. In 13.04 this is finalized to a SHA-256 content-addressed path; keep a `storage_path` column in metadata so the physical layout can change without breaking keys.

## Components

| Component | Purpose |
|---|---|
| `objects` upload handler | Consume request body as a stream to a temp file |
| `objects` download handler | Stream file to response with correct headers |
| `LocalFsBackend::{put_stream,get_stream}` | Bounded-buffer FS I/O + atomic rename |
| Temp + atomic rename | Crash-safe writes; no partial objects visible |

## Contracts

* `PUT /v1/buckets/{bucket}/objects/{key}` body=octet-stream → `201/200 {object}`; header `Content-Type` stored
* `GET /v1/buckets/{bucket}/objects/{key}` → `200` streamed bytes + `Content-Length`, `Content-Type`
* `HEAD /v1/buckets/{bucket}/objects/{key}` → `200` metadata headers, no body
* `404` when object/bucket absent (or cross-project)
* Extend `contracts/openapi/forge-storage.openapi.yaml`

## Data model

Uses `objects` table from 13.02; add `storage_path TEXT NOT NULL` (migration `0002`). Populate `size_bytes`, `content_type`, `updated_at` on upload.

## Configuration

```text
FORGE_STORAGE_STREAM_BUFFER_BYTES   # default 65536 (64 KiB chunk)
FORGE_STORAGE_MAX_OBJECT_BYTES      # optional hard cap; default unset (quota in 13.06)
```

## Security considerations

* Key path sanitized (no traversal) before mapping to `storage_path`
* Temp files created with restrictive permissions; cleaned on failure
* Project scope enforced on both PUT and GET

## Observability

* Log upload/download with `project_id`, `bucket`, `key`, `size_bytes`, duration
* Metric: `storage_upload_bytes_total`, `storage_download_bytes_total`, histogram of object sizes

## Failure handling

* Client disconnect mid-upload → temp file removed, no metadata row committed
* Disk write error → `500`, partial temp cleaned
* Download of missing object → `404`
* Overwrite is atomic: readers never see a half-written object

## Manual verification

```bash
BASE=localhost:4107; P='-H X-Forge-Project:proj-a'
head -c 50000000 /dev/urandom > /tmp/big.bin   # 50 MB
curl -fsS $P -XPUT --data-binary @/tmp/big.bin \
  -H 'content-type: application/octet-stream' \
  "$BASE/v1/buckets/artifacts/objects/big.bin"
curl -fsS $P "$BASE/v1/buckets/artifacts/objects/big.bin" -o /tmp/big.out
cmp /tmp/big.bin /tmp/big.out && echo "round-trip OK"
```

## Acceptance criteria

* Upload and download stream bytes with bounded memory (large file does not load fully into RAM)
* Round-trip of a large fixture is byte-identical
* Interrupted upload leaves no visible/partial object
* Overwrite is atomic; concurrent readers never see partial data
* Metadata records size and content-type
* OpenAPI updated and valid
