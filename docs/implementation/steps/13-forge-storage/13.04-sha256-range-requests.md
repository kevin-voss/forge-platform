# Step 13.04: SHA-256 + range requests

## Status

Not started

## Epic

`13-forge-storage` → [`../../epics/13-forge-storage.md`](../../epics/13-forge-storage.md)

## Goal

Compute and store a SHA-256 for every uploaded object (finalizing the content-addressed layout and enabling corruption detection), and support HTTP byte-range requests on download.

## Why this step exists

`specs.md` Step 13 requires that corrupted data is detected and that range requests work (resumable/partial downloads for large artifacts). SHA-256 also finalizes the content-addressed path that makes optional dedup possible.

## Dependencies

* `13.03` (streamed upload/download)

## Scope

* Compute SHA-256 during streamed upload (single pass, no re-read)
* Store hash in metadata; move object into content-addressed path `objects/<aa>/<hash>`
* Return `ETag`/`X-Content-SHA256` on GET/HEAD
* Verify integrity on download (detect on-disk corruption → error, not silent)
* `Range` request support (`bytes=start-end`, open-ended, multiple not required)
* `Accept-Ranges: bytes` and `206 Partial Content` responses

## Explicitly out of scope

* Signed tokens (13.05)
* Quotas + delete (13.06)
* Full dedup GC (documented optional)

## Architecture

```text
PUT: stream → hasher.update(chunk) + tmp write → finalize hash
     → move tmp to objects/<hash[0:2]>/<hash> (skip if identical hash exists = natural dedup)
     → metadata.storage_path = content-addressed path, sha256 = hash

GET (Range): parse Range → seek(start) → stream (end-start+1) bytes → 206
GET (verify): optional streaming re-hash mode or trailer check
```

## Components

| Component | Purpose |
|---|---|
| Streaming hasher | Single-pass SHA-256 during upload |
| Content-addressed mover | Place blob by hash; dedupe identical content |
| Range parser | Validate/clamp `Range` header |
| Integrity checker | Detect corrupted on-disk blobs |

## Contracts

* `PUT .../objects/{key}` response includes `sha256`; server may also honor client `X-Expected-SHA256` and reject mismatch with `422`
* `GET .../objects/{key}` returns `ETag: "<sha256>"`, `Accept-Ranges: bytes`
* `GET` with `Range: bytes=0-1023` → `206` + `Content-Range` header
* Invalid range → `416 Range Not Satisfiable`
* Corrupted blob detected → `500` with error code `integrity_error` (never silent success)
* Extend OpenAPI

## Data model

`objects.sha256` now NOT NULL after upload; `storage_path` points at content-addressed location. Add optional `refcount` column (migration `0003`) to support safe delete/dedup in 13.06.

## Configuration

```text
FORGE_STORAGE_VERIFY_ON_READ   # off|full (default off; full re-hashes on GET)
```

## Security considerations

* Reject client-supplied hash mismatch (prevents storing mislabeled content)
* Range bounds validated to avoid out-of-range reads

## Observability

* Log computed hash + dedup hit/miss on upload
* Metric: `storage_dedup_hits_total`, `storage_integrity_errors_total`, `storage_range_requests_total`

## Failure handling

* Client `X-Expected-SHA256` mismatch → `422`, object not stored
* On-disk hash mismatch at read (verify mode) → `500 integrity_error`
* Unsatisfiable range → `416`

## Manual verification

```bash
BASE=localhost:4107; P='-H X-Forge-Project:proj-a'
curl -fsS $P -XPUT --data-binary @/tmp/big.bin \
  "$BASE/v1/buckets/artifacts/objects/big.bin" | grep -o '"sha256":"[0-9a-f]*"'
LOCAL=$(shasum -a 256 /tmp/big.bin | awk '{print $1}')
SRV=$(curl -fsSI $P "$BASE/v1/buckets/artifacts/objects/big.bin" | tr -d '\r' | awk -F'"' '/ETag/{print $2}')
test "$LOCAL" = "$SRV" && echo "checksum OK"
curl -fsS $P -H 'Range: bytes=0-1023' \
  "$BASE/v1/buckets/artifacts/objects/big.bin" -o /tmp/range.bin -w '%{http_code}\n'
test "$(wc -c < /tmp/range.bin)" = 1024 && echo "range OK"
```

## Acceptance criteria

* Every object has a stored SHA-256; GET returns it as `ETag`
* Client-declared checksum mismatch is rejected (`422`)
* On-disk corruption is detected, never returned as success
* `Range` requests return `206` with correct bytes and `Content-Range`; invalid ranges → `416`
* Identical content is stored once (dedup hit observable)
* OpenAPI updated and valid
