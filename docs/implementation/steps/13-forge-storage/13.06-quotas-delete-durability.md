## Goal

Enforce per-project storage quotas, implement object delete (with dedup-safe refcounting and cascade bucket delete), and prove that all buckets, objects, and metadata survive a service restart.

## Why this step exists

`specs.md` Step 13 requires quotas, object deletion, and that storage survives restart. This step closes the durability and lifecycle gaps so the epic demo (13.07) can assert the full lifecycle including cleanup.

## Dependencies

* `13.05` (signed tokens) — all object operations available

## Scope

* Per-project byte quota (config default + optional per-project override table)
* Quota enforcement on upload (reject with `413` when exceeded)
* `DELETE .../objects/{key}` — remove metadata + decrement blob refcount; GC blob when refcount hits 0
* Cascade `DELETE /v1/buckets/{bucket}?force=true` — delete all objects then bucket
* Usage reporting: `GET /v1/usage` (bytes used vs quota per project)
* Restart durability verification (metadata + blobs persist)

## Explicitly out of scope

* Background async GC daemon (synchronous refcount GC is sufficient)
* Object versioning/soft-delete/trash
* Cross-project quota pooling

## Architecture

```text
Upload: current_usage + incoming_size > quota → 413 (before committing metadata)
Delete: metadata row removed → blob.refcount-- → if 0 unlink objects/<hash>
Usage:  SUM(size_bytes) per project (maintained incrementally + reconciled on boot)
```

Durability: metadata in SQLite (WAL) on the named volume; blobs in `objects/`. On boot, optionally reconcile usage counters from metadata to self-heal after crashes.

## Components

| Component | Purpose |
|---|---|
| Quota manager | Track/enforce per-project usage |
| Delete handler | Refcount-aware object + cascade bucket delete |
| Blob GC | Unlink content-addressed blob when unreferenced |
| Usage endpoint | Report bytes used vs quota |
| Boot reconciler | Recompute usage from metadata on startup |

## Contracts

* `DELETE /v1/buckets/{bucket}/objects/{key}` → `204` | `404`
* `DELETE /v1/buckets/{bucket}?force=true` → `204` (cascade); without force + non-empty → `409`
* `GET /v1/usage` → `200 { "project_id":"...", "used_bytes":N, "quota_bytes":M, "objects":K }`
* Upload exceeding quota → `413 quota_exceeded`
* Extend OpenAPI

## Data model

```sql
-- migration 0004
CREATE TABLE project_quota (
  project_id TEXT PRIMARY KEY,
  quota_bytes INTEGER NOT NULL
);
CREATE TABLE blobs (
  sha256 TEXT PRIMARY KEY,
  size_bytes INTEGER NOT NULL,
  refcount INTEGER NOT NULL DEFAULT 0
);
-- objects.sha256 references blobs.sha256
```

Default quota from `FORGE_STORAGE_DEFAULT_QUOTA_BYTES` when no `project_quota` row exists.

## Configuration

```text
FORGE_STORAGE_DEFAULT_QUOTA_BYTES   # default e.g. 1073741824 (1 GiB)
FORGE_STORAGE_RECONCILE_ON_BOOT     # true|false (default true)
```

## Security considerations

* Delete respects project scope (cannot delete another project's object)
* Force-cascade delete requires same project auth
* Quota override table writable only via admin/internal path (documented; enforcement with Identity roles later)

## Observability

* Log deletes with refcount before/after + GC decision
* Metric: `storage_used_bytes{project}`, `storage_quota_rejections_total`, `storage_blobs_gc_total`

## Failure handling

* Delete of missing object → `404`
* Quota exceeded → `413`, upload not committed, temp cleaned
* Crash mid-delete → boot reconciler restores consistent usage; orphan blobs GC'd on next reconcile
