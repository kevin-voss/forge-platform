## Goal

Make Forge Control **enforce** authentication and authorization: a middleware extracts the bearer token, introspects it against Identity, resolves the principal + project role, and calls Identity's `authz/check` (or applies the cached matrix) for each mutating action — rejecting unauthenticated (`401`) and unauthorized (`403`) requests. The `FORGE_AUTH_MODE=dev` bypass is removed from the default; `enforce` becomes the default mode.

## Why this step exists

This is the payoff of the epic and the acceptance-critical requirement of `specs.md` Step 09: "unauthenticated platform modifications are rejected" and "project roles are enforced." Until now Control mutated freely; this step wires the enforcement point using the Identity primitives from 09.03–09.05.

## Dependencies

* Step `09.03` — introspection
* Step `09.04` — authorization decision / matrix
* Step `09.05` — token principals
* Epic `02-forge-control` — the mutation endpoints being protected

## Scope

* Auth middleware in Control: parse `Authorization: Bearer`, introspect via Identity, attach principal + memberships to the call context
* Action mapping: map each Control mutation route to an `action` (e.g. `POST /v1/deployments` → `deployment.create`) and the relevant `project_id`
* Authorization: call Identity `authz/check` (with a short introspection cache) and enforce
* Mode switch: `FORGE_AUTH_MODE` default becomes `enforce`; `dev` remains an explicit opt-in for other demos (documented as insecure)
* Consistent `401`/`403` error bodies (reuse Control error format `02.06`)
* Read endpoints: require authentication but allow `viewer`+ ; mutations require role per matrix

## Explicitly out of scope

* Gateway-level auth (optional; not required)
* New roles/permissions (defined in 09.04)
* CLI login UX (09.07)

## Architecture

```text
request → AuthMiddleware:
  token = bearer(request) or → 401
  principal = Identity.introspect(token)  (cached ~10s)  or → 401 if inactive
  action, project = mapRoute(request)
  decision = Identity.authz/check(principal, project, action)  (cached)
  decision.allow ? proceed : 403
```

## Components

| Component | Responsibility |
|---|---|
| `AuthMiddleware` | Authenticate + authorize each request |
| `RouteActionMap` | Map (method, path) → action + project extractor |
| `IdentityClient` | Call introspect + authz/check with caching |
| `IntrospectionCache` | Short TTL cache honoring revocation |

## Contracts

* No new external API; Control endpoints now require `Authorization: Bearer <token>`
* Error bodies: `401 { error: "unauthenticated" }`, `403 { error: "forbidden", required_action, role }`
* Depends on Identity `POST /v1/auth/introspect` and `POST /v1/authz/check`

No new tables in Control. Cache is in-memory.

## Configuration

```text

```

## Security considerations

* Default is now secure (`enforce`); `dev` bypass must be set explicitly and logs a prominent warning on startup
* Introspection cache TTL is short so revocation (09.05) takes effect quickly; a revoke event can also bust the cache entry
* Tokens are never logged in Control (log principal id + action only)
* Fail-closed: if Identity is unreachable, mutations are rejected (503/401), not allowed through

## Observability

* Log every denied request `{principal, action, project, decision, reason}`
* Metrics: `forge_control_auth_total{result=ok|401|403}`, `forge_identity_introspect_latency_ms`, `forge_authz_cache_hits_total`
* Trace: middleware span linking to Identity call spans (distributed trace foundation for epic 12)

## Failure handling

* Missing/invalid token → `401`
* Valid token, insufficient role → `403`
* Identity unreachable → mutations `503` (fail-closed); reads may serve from cache within TTL then `503`
* `dev` mode → bypass with a loud warning; used only by pre-09 demos
