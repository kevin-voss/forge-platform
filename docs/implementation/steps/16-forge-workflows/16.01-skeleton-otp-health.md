## Status

## Goal

Stand up the `forge-workflows` Elixir/OTP application with a supervision tree, HTTP health endpoints, structured logging, graceful shutdown, a Docker image, and Compose wiring on host port `4302` — no workflow engine yet.

## Why this step exists

The durable engine needs a supervised OTP application and a contract-compliant HTTP surface before definitions and state are added. Isolating the skeleton keeps engine steps focused.

## Dependencies

* Epics `00`, `01` conventions (health, logs, `PORT`, SIGTERM)

## Scope

* Elixir app under `services/forge-workflows` (mix project + OTP application)
* HTTP server (Bandit/Plug or Phoenix) with `GET /health/live`, `GET /health/ready`, `GET /`
* Structured JSON logging + request IDs
* Supervision tree scaffold (registry/dynamic supervisor placeholders for runs)
* Dockerfile (Elixir release) + Compose service on `4302`
* Makefile targets; port reservation confirmed

## Explicitly out of scope
