## Goal

Add compensation (saga-style rollback): each forward step may declare a compensating action; on failure or on an approved rollback decision, the engine executes compensators in reverse order, including a `rollback deployment` action wired to Control/reconciliation.

## Why this step exists

`specs.md` Step 16 requires compensation actions and the demo rolls back a deployment after approval. This closes the loop from failure → diagnosis → approval → safe rollback.

## Dependencies

* `16.05` (durable approvals)
* Epic `02-forge-control` / `07-deployment-reconciliation` rollback API (minimum: deployment rollback endpoint)

## Scope

* Per-step compensator declaration in definitions
* Saga log: record executed forward steps + their compensators
* Rollback trigger: step failure (auto) or explicit `rollback` action after approval
* `control.rollback_deployment` action via a Control client
* Reverse-order compensation execution with idempotency
* Final report/result stored and emitted (optionally as an event/Storage artifact)

## Explicitly out of scope

* Demo assembly (16.07)
* New primitives (16.03)

## Architecture

```text
forward step done → append saga_log(step, compensator)
failure OR approved rollback → iterate saga_log in reverse → run each compensator (idempotent)
rollback action → ControlClient.rollback_deployment(deployment_id)
store final report → workflow_runs.result (+ optional Storage/Events)
```

## Components

| Component | Purpose |
|---|---|
| Saga log | Ordered record of compensable actions |
| `Compensator` runner | Reverse-order execution, idempotent |
| `ControlClient` | `rollback_deployment` + status |
| Report writer | Persist final report; optional Storage/Events |

## Data model

```sql

  id UUID PRIMARY KEY, run_id UUID REFERENCES workflow_runs(id),
  step_id TEXT NOT NULL, compensator TEXT NOT NULL, args JSONB,
  status TEXT NOT NULL,      -- pending|compensated|failed

```

## Security considerations

* Rollback is a privileged action; only executed after approval or defined auto-compensation
* Control client uses scoped credentials; project scope enforced
* Compensators idempotent so re-runs after crash don't double-act

## Observability

* Log each compensation (step, compensator, outcome) + final rollback result
* Metric: `workflow_compensations_total{status}`, `workflow_rollbacks_total{result}`

## Failure handling

* Compensator failure → recorded; continue remaining compensators; run ends `failed` with saga status
* Control rollback failure → retried per policy then surfaced in report
* Crash mid-compensation → resume reverse execution from saga log (idempotent)
