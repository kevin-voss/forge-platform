## Goal

Implement the core step primitives — retry (with policy/backoff), delay, timeout, parallel (fan-out/fan-in), and conditional (branching) — as durable, resumable building blocks the engine executes.

## Why this step exists

`specs.md` Step 16 lists these primitives and requires that failed actions retry per policy and that state is durable. These are the composable pieces every real workflow (including the demo) is built from.

## Dependencies

* `16.02` (durable run state)

## Scope

* `retry`: max attempts, backoff (fixed/exponential), retry-on predicate; attempt count persisted
* `delay`: durable timer (resumes correctly after restart, not by sleeping in memory)
* `timeout`: step-level and run-level deadlines
* `parallel`: run sub-steps concurrently, join, aggregate results/errors
* `conditional`: branch on prior step output/context
* Definition schema extensions for these step types

## Explicitly out of scope

* Event triggers + agent steps (16.04)
* Approval steps (16.05)
* Compensation (16.06)

## Architecture

```text
step.type ∈ {task, retry, delay, timeout, parallel, conditional}
retry:  attempt++ persisted; on failure schedule next attempt per backoff
delay:  persist wake_at; timer (or boot scan) resumes when due
parallel: spawn child tasks under supervisor; join; persist each child status
conditional: evaluate predicate over run context → choose next step_id
```

Durable timers: store `wake_at` in DB; a scheduler process (and boot scan) fires due delays so restarts don't lose or duplicate them.

## Contracts

Definition schema additions (YAML), e.g.:

```yaml
steps:
  - id: collect
    type: task
    action: noop
    retry: { max_attempts: 3, backoff: exponential, base_ms: 200 }
  - id: wait
    type: delay
    delay_ms: 5000
  - id: fanout
    type: parallel
    branches: [logs, metrics]
  - id: decide
    type: conditional
    when: "context.severity == 'high'"
    then: escalate
    else: close
```

`GET /v1/runs/{id}` reflects attempts, branch statuses, and timer waits.

## Security considerations

* Conditional predicate is a safe expression evaluator (no arbitrary code)
* Parallelism capped to avoid resource exhaustion

## Observability

* Log retries (attempt, backoff), delay scheduling, parallel join, branch decisions
* Metric: `workflow_step_retries_total`, `workflow_parallel_branches`, `workflow_timeouts_total`

## Failure handling

* Retry exhausted → step `failed` (may trigger compensation later)
* Timeout → step `failed` reason `timeout`
* Parallel: partial failure policy documented (fail-fast vs collect); default collect then fail if any required branch failed
* Delay survives restart (fired from persisted `wake_at`)
