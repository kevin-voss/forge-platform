## Goal

Introduce collections that fix a vector dimension and store records `{id, vector[dim], metadata}` durably, with collection CRUD and record read/list — enforcing dimension consistency and persisting across restart.

## Why this step exists

Search (17.03) needs a well-defined, dimension-consistent store with metadata. Establishing collections + records first isolates storage from query logic.

## Dependencies

* `17.01` (skeleton + persistence)

## Scope

* Collection APIs: create (with `dim`, distance=`cosine`), list, get, delete
* Record model: `id`, `vector` (len == collection dim), `metadata` (JSON), `document_ref` (optional Storage pointer)
* Durable persistence: vectors in a memory-mapped file per collection; metadata in SQLite
* Read/list records (get by id; list with pagination)
* Dimension enforcement on insert (17.03 does upsert; here define storage + validation)

## Explicitly out of scope

* Upsert + NN query (17.03)
* Namespaces/ACL (17.04)
* Models/Agents integration (17.05)

## Architecture

```text
collection = { name, dim, distance, count }
vectors/<collection>.vec   # fixed-stride f32 records (mmap)
meta/index.db              # collections table + records table (id → offset, metadata)
```

## Components

| Component | Purpose |
|---|---|
| `CollectionStore` | Create/list/get/delete collections |
| `VectorFile` | mmap fixed-stride vector storage |
| `MetaStore` (SQLite) | Record metadata + id→offset map |
| `collections`/`records` routes | REST |

## Contracts

* `POST /v1/collections` `{ "name","dim":384,"distance":"cosine" }` → `201 {collection}`
* `GET /v1/collections` / `GET /v1/collections/{name}` / `DELETE /v1/collections/{name}`
* `GET /v1/collections/{name}/records/{id}` → `200 {record}` | `404`
* `GET /v1/collections/{name}/records` → paginated list
* Extend OpenAPI

## Data model

```sql
CREATE TABLE collections (
  name TEXT PRIMARY KEY, project_id TEXT NOT NULL, dim INTEGER NOT NULL,
  distance TEXT NOT NULL, count INTEGER NOT NULL DEFAULT 0, created_at TEXT
);
CREATE TABLE records (
  collection TEXT NOT NULL, id TEXT NOT NULL, offset INTEGER NOT NULL,
  metadata TEXT, document_ref TEXT, deleted INTEGER DEFAULT 0,
  PRIMARY KEY (collection, id)
);
```

## Failure handling

* Vector length != collection dim → `422 dimension_mismatch`
* Duplicate collection → `409`
* Missing collection/record → `404`
* Corrupt vector file detected on open → error (not silent)
