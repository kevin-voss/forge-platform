## Goal

Scope every collection, record, upsert, and query to a project namespace derived from Identity, so that memory in one project is invisible and unqueryable from another, with access control enforced on all operations.

## Why this step exists

`specs.md` Step 17 requires namespaces, access control, and project data isolation. Retrieval must never leak one tenant's incidents into another's diagnosis.

## Dependencies

* `17.03` (upsert + query)
* Epic `09-forge-identity` for project scope (minimum: verified project id; `dev` header bypass allowed until enforced)

## Scope

* Project-scope middleware (Identity token or `X-Forge-Project` in dev mode)
* Enforce `project_id` on all collection/record/query operations
* Optional sub-namespaces within a project (e.g. `agent-memory`, `docs`) as a `namespace` field
* Cross-project access returns `404` (no existence leak)
* ACL check hook for read/write (role-aware when Identity enforced)

## Explicitly out of scope

* Models/Agents integration (17.05)
* Fine-grained per-record ACL beyond project+namespace
* Demo (17.06)

## Architecture

```text
request → scope middleware → project_id (+ optional namespace)
every store/query call filters by (project_id[, namespace])
cross-project id → 404
```

## Components

| Component | Purpose |
|---|---|
| Scope middleware | Extract/verify project + namespace |
| ACL hook | Read/write permission check |
| Scoped store calls | Mandatory project filter everywhere |

## Contracts

* All existing endpoints now require project context (`Authorization` or `X-Forge-Project`)
* Optional `?namespace=` on collection/query operations
* Cross-project access → `404`; unauthorized (enforced mode) → `401`/`403`
* Extend OpenAPI (parameters + auth)

## Data model

`collections.project_id` + new `namespace` column (migration `0002`); records inherit collection scope. Uniqueness becomes `(project_id, namespace, collection name)`.

## Configuration

```text
FORGE_AUTH_MODE       # dev|enforced
FORGE_IDENTITY_URL    # token verification when enforced
```

## Security considerations

* Deny-by-default; every query filtered by project (+namespace)
* No existence leak (404 not 403 for cross-project)
* Dev-mode header bypass logged as warning
* ACL check auditable

## Observability

* Log scope on each operation (project, namespace)
* Metric: `memory_acl_denied_total{reason}`

## Failure handling

* Missing project context (enforced) → `401`
* Cross-project access → `404`
* Unknown namespace → treated as empty (create-on-upsert optional, documented)
