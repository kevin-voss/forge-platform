## Goal

Add a convenience path where Memory embeds text via Forge Models before storing/querying, and register a `memory.search`/`memory.upsert` retrieval tool in Forge Agents so agents can store and cite memory records.

## Why this step exists

`specs.md` Step 17 integration: Models produces embeddings, Agents queries memory. This wires Memory into the AI loop so the demo (17.06) — and the capstone — can have an agent retrieve similar incidents.

## Dependencies

* `17.04` (namespaces/ACL)
* Epic `14-forge-models` `POST /embed` (`14.03`), with matching `embedding_dim`
* Epic `15-forge-agents` tool registry (`15.03`) + platform tools (`15.05`)

## Scope

* Memory-side: accept text + `model` on upsert/query and call Models to embed (in addition to raw-vector path)
* Dimension check: embedding dim must equal collection dim
* Agents-side: `memory.search` (read) + `memory.upsert` (write) tools with permissions + schemas
* `memory.search` returns records + scores usable in an agent's reasoning/citation

## Explicitly out of scope

* Demo assembly (17.06)
* New search algorithms (17.03)
* HNSW

## Architecture

```text
Memory:  POST /query {text, model, top_k} → ModelsClient.embed(text) → cosine query
         POST /upsert {items:[{id,text,metadata}], model} → embed → store
Agents:  tools/memory.py → memory.search(query_text, collection, top_k) → Memory /query
                          → memory.upsert(...) (write, permissioned)
```

## Components

| Component | Purpose |
|---|---|
| `ModelsClient` (memory) | Call Forge Models `/embed` |
| Text upsert/query | Embed-then-store / embed-then-query |
| Agents `memory.*` tools | Retrieval + store tools |
| Dim guard | Reject model/collection dim mismatch |

## Contracts

* Memory: `POST .../query` also accepts `{ "text":"...","model":"local-embed-small","top_k":5 }`
* Memory: `POST .../upsert` also accepts `{ "items":[{ "id","text","metadata" }],"model":"..." }`
* Agents tool `memory.search` input `{ collection, query, top_k }` → `{ results:[{id,score,metadata}] }`
* Agents tool `memory.upsert` input `{ collection, items }` (permissioned, non-destructive)
* Extend both OpenAPIs / tool listing

## Security considerations

* Embedding calls pass project scope; results stay project-scoped
* `memory.search` requires read permission; `memory.upsert` requires write permission
* Dim mismatch rejected to prevent corrupt collections

## Observability

* Log embed-then-store/query with model + dim + latency

## Failure handling

* Models unavailable → clear error (`embedding_backend_unavailable`); raw-vector path still works
* Dim mismatch (model dim != collection dim) → `422`
* Agent tool errors normalized (per 15.05)
