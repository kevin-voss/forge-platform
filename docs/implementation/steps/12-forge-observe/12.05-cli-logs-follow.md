# Step 12.05: CLI `forge logs --follow`

## Status

Not started

## Epic

`12-forge-observe` → [`../../epics/12-forge-observe.md`](../../epics/12-forge-observe.md)

## Goal

Add `forge logs` to the CLI with a `--follow` mode that live-tails a service's/deployment's logs via Forge Observe (SSE stream over Loki tail), supporting the same correlation filters (`--project`, `--deployment`, `--service`, `--request-id`, `--trace-id`, `--since`). Falls back to Runtime's log streaming (`04.05`) when Loki tail is unavailable.

## Why this step exists

`specs.md` Step 12 lists "CLI log tail" as a required capability. Live tailing is the everyday developer workflow for watching a deployment; this makes the log query API (12.04) interactive and streaming.

## Dependencies

* Step `12.04` — log query filters + Loki backing
* Epic `04-forge-runtime` — per-workload log streaming as a fallback (`04.05`)
* Epic `03-forge-cli` + `09.07` — CLI framework + auth token
* Epic `05-forge-gateway` — SSE proxy support (`05.06`) for streaming through the gateway

## Scope

* Observe streaming endpoint: `GET /v1/logs/stream` (SSE) applying the 12.04 filters and pushing new entries live
* `forge logs` command: point-in-time query (uses 12.04) and `--follow` (uses the stream)
* Filters mapped to flags; human table + `--json` line output
* Fallback: if Observe/Loki tail is unavailable and a single service is targeted, stream from Runtime (`04.05`)
* Graceful Ctrl-C, reconnect on transient stream drop, backpressure handling

## Explicitly out of scope

* Alerts (12.06)
* Historical export/download of logs (query API suffices)
* Multi-cluster tailing

## Architecture

```text
forge logs --deployment dpl_1 --follow
  → Observe GET /v1/logs/stream?deployment=dpl_1  (SSE)
  → print entries as they arrive
  (fallback) if stream unavailable & single service:
  → Runtime GET /v1/workloads/{name}/logs?follow=true  (04.05)
```

## Components

| Component | Responsibility |
|---|---|
| Observe `LogStreamRoute` | SSE tail applying filters |
| `forge logs` command | Query + `--follow` |
| stream client | Consume SSE, reconnect, render |
| Runtime fallback | Stream from Runtime when Loki tail down |

## Contracts

* `GET /v1/logs/stream?project=&deployment=&service=&request_id=&trace_id=&since=` → `text/event-stream` of log entries (same shape as 12.04 entries)
* Consumes Runtime `04.05` log stream as fallback
* No new CLI server API beyond the stream endpoint

## Data model

None (streaming). Uses correlation fields from 12.01/12.02.

## Configuration

```text
FORGE_OBSERVE_URL              # observe endpoint (from profile)
FORGE_LOGS_RECONNECT_MS        # stream reconnect backoff (default 1000)
FORGE_LOGS_FALLBACK            # observe|runtime|auto (default auto)
```

## Security considerations

* Stream honors the same project-access authorization as 12.04 (enforced token scoping)
* Auth token attached to the SSE request; never printed
* Rendered log lines may contain already-masked secrets; CLI does not unmask

## Observability

* CLI prints a discreet status line on reconnect (stderr) so stdout stays clean log content
* Observe stream endpoint metrics: `forge_log_streams_active` gauge, `forge_log_stream_reconnects_total`
* Trace span `observe.logs.stream`

## Failure handling

* Stream drops → CLI reconnects with backoff, resuming from the last-seen timestamp (no gap/dupe where possible)
* Observe/Loki tail unavailable → fall back to Runtime stream for a single-service target; if not possible, clear error
* Ctrl-C → clean shutdown, flush, exit 0
* `403`/`401` → clear message, non-zero exit (from 09.07 mapping)
