## Goal

Assemble `demos/12-observability`: drive a request along `CLI → Control → Build → Runtime → Gateway → demo application` and prove that a **single trace ID** spans the whole path, its logs are queryable by project/deployment/trace, `forge logs --follow` tails it live, and an induced error fires the error-rate alert. Wire into `make demo DEMO=12` as the epic acceptance gate.

## Why this step exists

The observability pieces (instrumentation, dashboards, log query, tail, alerts) only prove the epic when a single command demonstrates one end-to-end distributed trace plus correlated logs and a firing alert against the real stack. This is the gate in `MASTER_PLAN.md`.

## Security considerations

## Observability

* `run.sh` prints the captured trace ID + per-service span list; on failure dumps Observe logs + Tempo query output

## Failure handling

* Trace not spanning all services → demo fails with the span list
* Correlated logs missing → demo fails
* Alert not firing on induced error → demo fails
* Cleanup trap tears down Compose on exit
