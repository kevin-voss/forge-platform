## Goal

Prove Forge Infrastructure can create and delete runtime nodes through a provider adapter while the scheduler treats those nodes exactly like any other node. The default gate uses the Docker provider as a local cloud simulation; optional billable paths exercise Hetzner, AWS, and Azure.

## Scope

* Local Docker `NodePool` creates multiple runtime-node containers.
* Nodes bootstrap, join Forge Network, register with Control, and become schedulable.
* Workloads deploy onto provider-created nodes.
* Draining and deletion remove nodes without orphaned provider resources.
* Optional provider demos behind explicit `FORGE_DEMO_TARGET`.

## Demo flow

```text
Create NodePool provider=docker minNodes=2
-> Infrastructure starts two runtime-node containers
-> each node joins the Forge network
-> scheduler places application replicas on them
-> scale NodePool to 3
-> third node appears and becomes schedulable
-> drain one node
-> workloads move
-> provider deletes the node
```
