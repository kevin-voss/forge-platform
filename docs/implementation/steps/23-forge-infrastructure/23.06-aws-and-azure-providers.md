## Goal

Implement AWS EC2 and Azure VM provider adapters using only primitive infrastructure resources: VMs, disks, private networks, public IPs, security primitives, and pricing metadata. No managed compute, database, queue, or container services are required.

## Why this step exists

Forge's portability promise depends on the same `NodePool` and application manifest working across major clouds. Hetzner proves the simpler cloud case in `23.05`; this step proves the abstraction also holds on AWS and Azure, where the temptation to depend on managed services is highest.

## Dependencies

* `23.01`-`23.03` - provider interface, ledger, node bootstrap.
* `23.05` - rate-limit, idempotency, orphan-reconciliation patterns from the first real cloud adapter.
* Epic `10-forge-secrets` - cloud credentials via secret refs.

## Scope

* `aws` adapter using EC2 instances, EBS volumes, VPC/subnets/security groups, Elastic IPs.
* `azure` adapter using Virtual Machines, managed disks, VNets/subnets/network security groups, public IPs.
* Operation-id idempotency via provider tags.
* Cloud-init bootstrap identical in shape to Hetzner.
* Provider-specific rate-limit/backoff handling.
* Orphan reconciliation based on Forge-managed tags.
* Pricing lookup sufficient for `getPricing()` and future cost-aware autoscaling.

## Explicitly out of scope

* EKS, ECS, RDS, SQS, Lambda, CloudWatch, App Service, Service Bus, Azure Database.
* Cross-provider cluster demo; that is a future non-gating demo after M1.
* Reserved capacity and spot/preemptible support.

## Observability

* Metrics `forge_infra_aws_api_requests_total{operation,status}` and `forge_infra_azure_api_requests_total{operation,status}`.
* Log per provider call with `{provider, operation, op_id, status, retry_count}`.
* Trace spans `infra.provider.aws.*` and `infra.provider.azure.*`.

## Failure handling

* Retried create operations search by `forge.op_id` tag before creating.
* Partially created network/disk/IP resources are recorded in the operation ledger and cleaned during compensation.
* Credential or quota failures mark the `NodePool` condition `ProviderBlocked`, not `Failed` for individual workloads.

## Implementation tasks

1. Add AWS adapter implementing all provider methods with EC2 primitives.
2. Add Azure adapter implementing all provider methods with VM primitives.
3. Add tag-based idempotency and orphan reconciliation for both adapters.
4. Add pricing lookups and capacity normalization into Forge CPU/memory/GPU units.
5. Register both providers and add recorded-fixture tests.
6. Document required IAM/RBAC permissions with least-privilege examples.
