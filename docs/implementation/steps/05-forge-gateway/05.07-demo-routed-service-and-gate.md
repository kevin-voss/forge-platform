## Goal

Deliver `demos/05-routed-service`, which deploys Go, Rust, and Python demo workloads and exposes them through `go.demo.localhost`, `rust.demo.localhost`, and `python.demo.localhost` via the gateway on port `4000`, proving stable-hostname routing, health-aware traffic, dynamic route updates without restart, and request-id propagation — the epic acceptance gate.

## Why this step exists

The spec defines a concrete multi-language routing demo and acceptance criteria (reachable without knowing runtime ports, no traffic to unhealthy workloads, route changes without restart, request IDs propagate, demo passes). This step composes all prior gateway steps into one falsifiable end-to-end check.

## Dependencies

* Prior steps in this epic: `05.01`–`05.06` (all)
* Cross-epic prerequisites: Runtime (epic 04) to run workloads; Control (epic 02) for endpoint data; demo images Go/Rust/Python from epic 01; `make demo` runner

## Scope

* `demos/05-routed-service/` with `README.md` and `run.sh`
* Script: bring up Control + Runtime + Gateway; deploy Go/Rust/Python workloads; wait for routes; curl each hostname through the gateway; verify request-id propagation; stop one workload and assert it drops out; redeploy/change a route and assert update without restart
* `make demo DEMO=05` wiring; epic acceptance

## Explicitly out of scope

* Source builds (epic 06)
* Rolling updates/rollback (epic 07)
* TLS / real DNS

## Architecture

```text
make demo DEMO=05
   │
   ▼
demos/05-routed-service/run.sh
   ├── compose up postgres control runtime gateway (+ registry)
   ├── deploy go/rust/python demos (via Control→Runtime)
   ├── wait for gateway routes (sync/refresh)
   ├── curl -H 'Host: go.demo.localhost' … (and rust/python)  → 200 + correct language
   ├── assert X-Request-Id propagates to upstream
   ├── stop rust workload → assert 503/dropped without restart
   └── change/redeploy → assert route update without gateway restart
```

## Security considerations

* Documents dev auth bypass + Docker socket as dev-only.
* Uses `Host` header injection in CI to avoid DNS dependencies.

## Observability

* `run.sh` prints each phase + pass/fail; on failure dumps gateway routes, gateway/runtime logs, and `docker ps`.

## Failure handling

* Any assertion failure → non-zero exit with context.
* Route not present in time → fail with a dump of `/admin/routes`.
* Cleanup on exit (trap): delete deployments, `compose down`.

## Acceptance criteria

* Go, Rust, and Python demos are reachable through the gateway by hostname without callers knowing runtime ports.
* Stopping a workload removes it from rotation (no traffic to unhealthy upstreams).
* A route change / redeploy takes effect without restarting the gateway.
* Request IDs propagate to upstream applications.
* Demo 05 exits `0`.
