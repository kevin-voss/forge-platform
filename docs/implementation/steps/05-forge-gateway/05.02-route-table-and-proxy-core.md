## Goal

Implement the in-memory route table and the reverse-proxy core: match incoming requests by host and/or path to a configured upstream and proxy them, with round-robin balancing across a route's upstreams. Routes are settable via a static config / admin endpoint (dynamic Control sync arrives in 05.03).

## Why this step exists

The gateway's reason to exist is proxying. Building the route table + proxy core against statically configured upstreams first lets us prove matching and forwarding before adding the moving parts (sync, health-awareness).

## Dependencies

* Prior steps in this epic: `05.01` (bootable service + health)
* Cross-epic prerequisites: a reachable upstream (any HTTP server; the Runtime-published demo port for manual tests)

## Scope

* Route model: `{host?, pathPrefix?, upstreams:[{url}], strategy: round_robin}`
* Host-based and path-based matching (host wins; path prefix as tiebreaker/fallback)
* `httputil.ReverseProxy`-based forwarding
* Round-robin selection across a route's upstreams
* An internal admin API to load/replace routes (`GET/PUT /admin/routes`) for testing and manual use
* Concurrency-safe route table (atomic swap on update)

## Explicitly out of scope

* Syncing routes from Control/Runtime (05.03)
* Skipping unhealthy upstreams (05.04)
* Request-id/forwarded headers/timeouts (05.05)
* WebSocket/SSE (05.06)

## Architecture

```text
request → match(host, path) → route → pick upstream (round-robin) → ReverseProxy → upstream
                              │
                        route table (RWMutex / atomic.Value)
                              ▲
                     PUT /admin/routes (replace snapshot)
```

## Components

| Component | Purpose |
|---|---|
| `internal/routes` | Route model + matcher + thread-safe table |
| `internal/proxy` | ReverseProxy wrapper + round-robin picker |
| admin routes | `GET/PUT /admin/routes` |
| main handler | dispatch: match → proxy, else 404 |

## Contracts

```text
Admin (internal):
  GET  /admin/routes            → 200 current route snapshot (json)
  PUT  /admin/routes            → 200 replace routes; body: [{host,pathPrefix,upstreams,strategy}]

Data plane:
  ANY  <matched host/path>      → proxied to upstream
  no match                      → 404 {error:{code:"no_route",...}}
```

## Data model

In-memory only:

```text
Route{ host string, pathPrefix string, upstreams []Upstream, strategy string }
Upstream{ url string }   // e.g. http://127.0.0.1:49173
```

## Configuration

```text
FORGE_GATEWAY_STATIC_ROUTES=/etc/forge/routes.json   # optional bootstrap routes
```

## Security considerations

* Admin route is internal; documented as unauthenticated in dev, to be protected in epic 09.
* Validate upstream URLs (scheme/host) before adding to the table.
* Prevent open-proxy behavior: only proxy to configured upstreams, never arbitrary client-specified targets.

## Observability

* Log each proxied request: matched route, chosen upstream, status, duration.
* Log route-table replacements (count of routes).

## Failure handling

* No matching route → `404` with error envelope (`no_route`).
* Upstream connection error → `502` (health-aware skipping comes in 05.04).
* Invalid `PUT /admin/routes` body → `400`.

## Implementation tasks

1. Define the route model + upstream types.
2. Implement the matcher (host first, then path prefix) with deterministic precedence.
3. Implement a thread-safe route table with atomic replace.
4. Implement the reverse-proxy wrapper + round-robin picker.
5. Add the main dispatch handler (match → proxy, else 404).
6. Add `GET/PUT /admin/routes` and optional static bootstrap from file.

## Acceptance criteria

* Requests are matched by host and/or path prefix and proxied to the configured upstream.
* Round-robin distributes across multiple upstreams.
* Routes can be replaced at runtime via `PUT /admin/routes` (atomic, thread-safe).
* No-match returns a `404` error envelope; the gateway never proxies to arbitrary client-supplied targets.
* Upstream connection errors surface as `502`.
