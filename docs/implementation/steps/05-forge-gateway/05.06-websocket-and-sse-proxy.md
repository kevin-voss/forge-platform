## Goal

Support long-lived connections through the gateway: proxy WebSocket upgrades (bidirectional) and Server-Sent Events (SSE) streams to upstreams without buffering or premature timeouts, while preserving request-id and forwarded headers from 05.05.

## Why this step exists

The spec lists WebSocket and SSE support as gateway features, and later product traffic (and log/event streams) depends on them. Naive reverse proxying with response buffering or a hard request timeout breaks streaming; this step handles those cases explicitly.

## Dependencies

* Prior steps in this epic: `05.02` (proxy core), `05.05` (headers/timeouts + streaming exemption flag)
* Cross-epic prerequisites: an upstream that speaks WS/SSE (a demo echo upstream is acceptable per the epic open question)

## Scope

* Detect and handle `Upgrade: websocket` requests: hijack the connection, dial the upstream, and copy bytes both directions until close
* SSE: detect `Accept: text/event-stream` / streaming responses; disable response buffering and flush chunks immediately
* Exempt streaming routes from the overall request timeout (05.05 flag) while keeping idle/read timeouts sane
* Propagate request-id + forwarded headers on the upgrade/stream

## Explicitly out of scope

* WebSocket message inspection/transformation (pass-through only)
* Backpressure tuning beyond sane buffer sizes
* Protocol translation (e.g. WS↔SSE)

## Architecture

```text
if Upgrade: websocket:
   hijack client conn ↔ dial upstream ↔ bidirectional copy until either closes
elif SSE (text/event-stream):
   proxy with Flusher; no buffering; stream events as they arrive
else:
   normal ReverseProxy (05.02/05.05)

streaming routes: overall timeout disabled; idle/read timeouts apply
```

## Security considerations

* Only upgrade/stream to configured upstreams (no open relay).
* Enforce sensible idle timeouts to avoid connection exhaustion.
* Preserve origin/host headers as configured; do not leak internal upstream addresses to clients.

## Observability

* Log WS/SSE connection open/close with duration and bytes (approx) + request id.
* Count active streaming connections (metric later).

## Failure handling

* Upstream refuses upgrade → return `502` before hijack completes.
* Either side closes → clean up both connections promptly (no goroutine leak).
* SSE upstream ends → close the client stream gracefully.
* Idle beyond timeout → close with a logged reason.

## Implementation tasks

1. Detect WebSocket upgrade requests and implement hijack + upstream dial + bidirectional copy.
2. Detect SSE responses and stream with immediate flushing (no buffering).
3. Wire the streaming-route timeout exemption from 05.05; apply idle/read timeouts.
4. Propagate request-id + forwarded headers on the upgrade/stream request.
5. Ensure symmetric cleanup on close from either side.
6. Add `FORGE_WS_ENABLED`/`FORGE_SSE_ENABLED` toggles.
