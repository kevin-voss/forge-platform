## Goal

Deliver the first production-shaped reference workload in Go that fully implements the runtime contract, plus the `demos/01-container-runtime` scaffold and Compose wiring for that one service.

## Why this step exists

Go is the earliest vertical slice: one real container, contract validator against it, and a demo path that later languages extend. This unblocks incremental language ports without waiting for all five.

## Dependencies

* `01.01` — contract definition
* `01.02` — shared validator to assert compliance

## Scope

* `demos/01-container-runtime/` demo README + `run.sh` (Go-only success is enough)
* Go app under e.g. `demos/01-container-runtime/apps/go/` (or `demo-go-api/`)
* Dockerfile, listens on `PORT`, health + `/`, structured JSON logs, SIGTERM handling
* Compose file or Compose overlay for the Go service on host port **4201**
* Invoke `tools/contract-validator` from demo/`run.sh` or Make
* No Forge platform SDK imports

## Explicitly out of scope

* Python / Kotlin / Rust / Elixir apps (later steps)
* Platform services, gateway routing, identity
* Mandatory OTEL export

## Architecture

```text
make demo DEMO=01

demos/01-container-runtime/run.sh
        │
        ├── docker compose up demo-go-api
        └── contract-validator --base-url http://127.0.0.1:4201 ...
```

## Security considerations

* Distroless/scratch or minimal base image preferred
* Run as non-root in container if practical

## Observability

* Structured JSON logs on stdout with fields from the log schema
* Request logging optional for `/`

## Failure handling

* Missing/invalid `PORT` → exit non-zero
* SIGTERM → finish in-flight (if any) and exit 0 within 10s
* Readiness may mirror liveness for this minimal app (both 200 after listen)
