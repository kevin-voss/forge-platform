## Goal

Ship a language-agnostic CLI/tool that validates a running HTTP workload against the runtime contract from `01.01`, so every demo language can be checked the same way.

## Why this step exists

Acceptance for Step 01 requires a shared test runner. Building it before (or with) the first demo language keeps language implementations honest and avoids five bespoke smoke scripts.

## Dependencies

* `01.01` — documented contract + OpenAPI/examples available

## Scope

* Implement `tools/contract-validator` (preferred home; may be a shell + Go/Python script suite)
* Checks:
  * process/container listens on configured `PORT` / base URL
  * `GET /health/live` → 200
  * `GET /health/ready` → 200
  * `GET /` → 200 JSON with `service`, `language`, `status`
  * required env vars are documented for callers (runner accepts base URL + expected identity)
  * structured log sample validation (stdin file or captured log path) against log schema
  * graceful shutdown helper: send SIGTERM to a PID or `docker stop` and assert exit within grace window
* Make target: `make contract-validate BASE_URL=...` or `tools/contract-validator` README usage
* Unit tests with a tiny fixture HTTP server (can live under `tools/contract-validator` or `tests/contracts/`)

## Explicitly out of scope

* Building the five demo applications
* Full platform CI matrix for all languages (wire in later steps)
* OpenAPI codegen / SDKs
* Proving OTEL export

## Architecture

```text
contract-validator
  ├── http checks  → BASE_URL/health/live|ready|/
  ├── log checks   → JSON lines vs schema
  └── shutdown     → SIGTERM / docker stop + timeout
```

The runner talks **only** over HTTP + OS signals/Docker. It must not import product SDKs.

## Contracts

Consumes artifacts from `01.01`:

* OpenAPI paths/status codes (informal assertion is enough; full schema validation optional)
* `contracts/examples/runtime-log.schema.json`

## Data model

CLI inputs (illustrative):

```text
--base-url http://127.0.0.1:4201
--expect-service demo-go-api
--expect-language go
--log-file /tmp/app.jsonl          # optional
--shutdown-pid <pid>               # optional
--shutdown-timeout 10s
```

Exit codes: `0` pass, non-zero fail with actionable stderr.

## Configuration

* No Forge platform DB/NATS required
* Grace timeout default **10s** (aligned with contract doc)

## Security considerations

* Tool is local-dev only; no network scanning beyond provided URL
* Do not require credentials for health/info in epic 01

## Observability

Validator prints a concise pass/fail report; structured logs optional.

## Failure handling

* Connection refused → fail with “service not listening”
* Wrong JSON shape → fail naming missing fields
* Shutdown timeout → fail (simulates forced kill case)
