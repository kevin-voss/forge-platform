## Goal

Add a Kotlin (JVM) workload implementing the runtime contract on host port **4202**, validated by the shared runner and included in demo 01.

## Why this step exists

Kotlin is the suggested language for Forge Control. A contract-compliant Kotlin demo proves JVM services can meet the same container rules before Control is built.

## Dependencies

* `01.03` — demo scaffold (Compose + run.sh patterns)
* `01.04` optional but recommended so the demo suite already covers two languages; **hard dependency is `01.03` only**

## Scope

* Kotlin app under `demos/01-container-runtime/apps/kotlin/`
* Dockerfile (multi-stage build OK)
* Required HTTP surface, structured logs, graceful shutdown (JVM shutdown hooks / Ktor application stop)
* Compose service on host port **4202**
* Extend demo 01 validation list
* Minimal framework (Ktor/Netty or similar); no Forge SDK

## Explicitly out of scope

* `services/forge-control` implementation
* Rust / Elixir demos
* GraalVM native-image requirement
