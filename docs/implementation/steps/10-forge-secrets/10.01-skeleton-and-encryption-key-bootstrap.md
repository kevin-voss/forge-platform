## Goal

Create the Forge Secrets service: a Rust HTTP service on host port `4104` satisfying the runtime contract (health, structured logs, env config, graceful shutdown), with a cryptographic bootstrap that loads a master key from the environment and can generate/wrap per-project data keys. No secrets are stored yet — this step establishes the service and key hierarchy.

## Why this step exists

Encrypting secrets (10.02) requires a working key hierarchy and a running service with Postgres. Building the skeleton + `KeyProvider` seam first isolates the crypto foundation and keeps the master-key handling auditable before any secret data exists.

Key hierarchy:

```text
FORGE_SECRETS_MASTER_KEY (env, 32 bytes)
   └─ wraps ─▶ per-project data key (stored wrapped in DB)
                  └─ encrypts ─▶ secret values (10.02)
```

## Components

| Component | Responsibility |
|---|---|
| `KeyProvider` (trait) | Supply the master key; seam for KMS/HSM later |
| `EnvMasterKeyProvider` | Load + validate master key from env |
| `DataKey` | Generate random data key; wrap/unwrap with master key (AEAD) |
| Health/identity routes | Contract compliance |

## Data model

```sql
CREATE TABLE project_data_keys (
  project_id     TEXT PRIMARY KEY,
  wrapped_key    BYTEA NOT NULL,     -- data key encrypted by master key
  key_version    INT  NOT NULL DEFAULT 1,

```

## Security considerations

* Master key comes only from env (or the `KeyProvider` seam); never persisted, never logged
* Wrapped data keys are stored; plaintext data keys live only in memory
* Startup fails fast (not ready) if the master key is missing or the wrong length — never runs in an insecure "no encryption" mode
* AEAD with per-operation nonces; document nonce management

## Observability

* Log startup key self-check result (`master_key_id`, ok/fail) — no key material
* Metric: `forge_secrets_ready` gauge; `forge_data_keys_total`
* Trace spans on wrap/unwrap (no plaintext in span attributes)

## Failure handling

* Missing/invalid master key → `/health/ready` non-200; refuses to serve secret ops
* DB unreachable → not ready; retries with backoff
* Wrap/unwrap round-trip failure at startup → not ready (fatal config error surfaced)
