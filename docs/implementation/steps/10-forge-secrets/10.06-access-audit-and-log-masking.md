## Goal

Record an **audit trail** of every secret access (set, reveal/resolve, rotate, delete, config change) with who/what/when/where, and enforce **log masking** so configured secret values never appear in Secrets' logs — publishing a masking convention/library other platform services adopt.

## Why this step exists

`specs.md` Step 10 requires "access audit", "masking in logs", and acceptance that "logs mask configured secret values" and "secrets are not logged." Auditing makes secret usage traceable (security + incident response); masking prevents accidental disclosure through the observability pipeline.

## Dependencies

* Step `10.02` — `:access` reveal hook + secret store
* Step `10.03` — authenticated principal for audit attribution
* Step `10.04` — resolve (deploy-time access) to be audited

## Scope

* `audit_events` table: action, principal, project/env, secret/config name, version, timestamp, source (IP/service), result — **never the value**
* Audit hooks on set, rotate, delete, `:access`, resolve, config changes
* Audit query API: `GET /v1/projects/{pid}/audit?name=&action=` (authorized)
* Log masking: a formatter/filter that replaces known secret values with `***` in Secrets' log output; a small masking library other services can import
* Masking without storing plaintext elsewhere: match against in-memory known values loaded per request, or value-hash comparison, so masking itself does not create a plaintext copy

## Explicitly out of scope

* Forwarding audit to the Events service (epic 11) — audit is DB-local here (seam noted)
* Platform-wide automatic masking rollout to every service (this provides the library + Secrets' own masking; other services adopt incrementally)
* Long-term audit retention/archival policy

## Architecture

```text
every secret access → AuditRecorder.record(event)   # value excluded by construction
log line → MaskingFilter → replace known secret values with ***
audit query → GET /audit (authorized, project-scoped)
```

## Data model

```sql
CREATE TABLE audit_events (
  id           BIGSERIAL PRIMARY KEY,
  at           TIMESTAMPTZ NOT NULL DEFAULT now(),
  project_id   TEXT NOT NULL,
  environment  TEXT,
  action       TEXT NOT NULL,          -- secret.set|secret.access|secret.rotate|secret.delete|config.set|resolve
  principal    TEXT NOT NULL,          -- user/service-account id
  name         TEXT,
  version      INT,
  result       TEXT NOT NULL,          -- ok|denied|error
  source       TEXT                    -- caller ip / service id
);
CREATE INDEX idx_audit_scope ON audit_events(project_id, at);
```

## Security considerations

* Audit records exclude secret values by construction (schema has no value column)
* Masking compares against values only in-memory for the current request/log line; no new plaintext store is created
* Audit query is project-scoped + authorized (a project cannot read another's audit)
* Denied access attempts are audited (result=denied) for intrusion detection

## Observability

* The masking filter is itself part of observability: verify no value leaks into stdout logs
* Metrics: `forge_audit_events_total{action,result}`, `forge_masked_log_lines_total`
* Trace spans carry names/actions, never values

## Failure handling

* Audit write failure → the underlying operation still succeeds but logs a masked warning + increments an error metric (audit is best-effort, not a denial-of-service vector); configurable strict mode can fail-closed
* Masking filter must never crash the logger; on error it errs toward redaction (drops the line's dynamic content)
* Audit query with no scope/authorization → `403`

## Manual verification

```bash
make dev
P=prj_1; E=production
curl -s -X PUT localhost:4104/v1/projects/$P/envs/$E/secrets/DATABASE_PASSWORD -H "Authorization: Bearer $DEV" -d '{"value":"pw-secret"}' -H 'content-type: application/json'
curl -s -X POST localhost:4104/v1/projects/$P/envs/$E/secrets/DATABASE_PASSWORD:access -H "Authorization: Bearer $DEV" >/dev/null
curl -s "localhost:4104/v1/projects/$P/envs/$E/audit?name=DATABASE_PASSWORD" -H "Authorization: Bearer $DEV" | jq '.[] | {action, result, has_value: (has("value"))}'
# no plaintext in logs
docker logs forge-secrets 2>&1 | grep -c 'pw-secret' | grep -q '^0$' && echo masked-ok
```
