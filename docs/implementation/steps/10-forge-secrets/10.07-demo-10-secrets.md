## Goal

Assemble `demos/10-secrets`: set a secret + config, deploy a demo app that reports whether the secret is present (never the value), rotate the secret and redeploy so the app sees the new value, verify list shows metadata only, and verify logs mask secret values. Wire into `make demo DEMO=10` as the epic acceptance gate.

## Why this step exists

The secrets pieces (encryption, config, isolation, injection, CLI, audit/masking) only prove the epic when a single command demonstrates the end-to-end lifecycle against the real secured stack. This is the gate in `MASTER_PLAN.md`.

## Dependencies

* Steps `10.01`–`10.06` — full secrets service + injection + CLI + audit/masking

## Scope

* `demos/10-secrets/` Compose overlay: Identity + Control + Secrets + Runtime + demo app
