## Goal

Deliver the `demos/02-control-plane` demo that boots Control + PostgreSQL, creates the full project → environment → application → service → deployment hierarchy over HTTP, reads it back via `expand=tree`, asserts the shared error format and restart durability, and serves as the epic acceptance gate.

## Why this step exists

The spec defines a concrete demo for Step 02 and acceptance criteria (state survives restart, stable UUIDs, migrations run, invalid relationships rejected, shared error format, OpenAPI exists, demo passes). This step wires all prior steps into one runnable, falsifiable end-to-end check.

## Dependencies

* Prior steps in this epic: `02.01`–`02.07` (all)
* Cross-epic prerequisites: epic `00` Compose stack + Postgres; `make demo` runner pattern

## Scope

* `demos/02-control-plane/` with `README.md`, `run.sh`, and any Compose overlay
* Script that: starts Control + Postgres, runs migrations, executes the create chain, reads back the tree, checks an error case, restarts Control, and re-reads to prove durability
* `make demo DEMO=02` wiring
* Epic acceptance: assert all Step 02 criteria

## Explicitly out of scope

* CLI usage (that is `demos/03-cli-control`)
* Container execution of the deployed image (epic 04)
* Any new API endpoints

## Architecture

```text
make demo DEMO=02
   │
   ▼
demos/02-control-plane/run.sh
   ├── docker compose up postgres forge-control
   ├── wait for /health/ready
   ├── POST project → environment → application → service → deployment
   ├── GET /v1/projects/{id}?expand=tree  → assert full hierarchy
   ├── GET unknown id → assert error envelope + 404
   ├── docker restart forge-control → wait ready
   └── re-GET tree → assert identical ids (durability)
```

## Observability

* `run.sh` prints each step and a final success/failure summary.
* On failure, dumps `docker logs forge-control` tail for triage.

## Failure handling

* Any failed assertion → non-zero exit with a clear message.
* Readiness timeout → fail with logs.
* Cleanup (`docker compose down` / schema reset) runs on exit (trap).
