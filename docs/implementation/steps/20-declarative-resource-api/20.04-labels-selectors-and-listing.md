## Goal

Turn the unfiltered, unpaginated `GET` list from `20.02` into a real listing API: label/annotation validation, `labelSelector` query support (equality, `in`/`notin`, existence/non-existence), field filters (`phase`, name prefix), stable cursor pagination, and a list envelope that carries the snapshot `resourceVersion` a subsequent watch can start from.

## Why this step exists

`20.05`'s watch needs a well-defined `since={resourceVersion}` starting point, and that only means something if `LIST` itself reports the `resourceVersion` of the snapshot it returned. Label selectors are how every later kind (Node pools by region, Applications by team, Deployments by canary tag) gets queried without a bespoke filter parameter per field — building the general selector grammar once here is what lets `21`–`43` skip reinventing it.

## Dependencies

* `20.02` — generic `GET` list (unfiltered baseline)
* `20.01` — `labels`/`annotations` columns, `resources_kind_scope_idx`

## Scope

* Label/annotation key/value validation: keys `[a-z0-9]([-a-z0-9_.]*[a-z0-9])?` (optional `prefix/` segment ≤ 253 chars), values ≤ 63 chars — rejected at write time (`20.02`'s create/replace/patch paths call into this step's validator)
* `labelSelector` query grammar: `key=value`, `key!=value`, `key in (a,b,c)`, `key notin (a,b,c)`, bare `key` (existence), `!key` (non-existence); comma-separated terms are AND-ed
* Field filters: `?phase=Ready`, `?namePrefix=invoice-`
* Cursor pagination: opaque base64 cursor over `(name, id)`, stable `ORDER BY name ASC, id ASC`, `?limit=` (default 50, max from `FORGE_LIST_MAX_PAGE_SIZE`)
* List envelope: `{ apiVersion, kind: "<Kind>List", resourceVersion, items: [...], nextCursor }`
* GIN indexes on `labels`/`annotations` and a covering btree for cursor ordering

## Explicitly out of scope

* Watch (`20.05` consumes this step's `resourceVersion` semantics but the stream itself is next)
* Ownership-based filtering (e.g. "list resources owned by X") — deferred to `20.06` if needed, not required for the epic gate
* Full JSONPath/field-selector expressiveness beyond `phase`/`namePrefix` — additional field filters are added by the kind that needs them, not by this generic step

## Architecture

```text
GET .../{plural}?labelSelector=env=prod,tier!=cache&phase=Ready&namePrefix=inv-&limit=50&cursor=...
        │
        ▼
LabelSelectorParser → SQL predicate over labels JSONB
FieldFilter          → SQL predicate over status->>'phase' / name
CursorCodec          → decode (name, id) → WHERE (name, id) > (?, ?)
        │
        ▼
{ resourceVersion: max(resource_version) over the matched set at query time,
  items: [...], nextCursor }
```

Selector grammar (BNF-ish):

```text
selector    := term (',' term)*
term        := existence | negatedExistence | equality | inequality | setIn | setNotIn
existence   := key
negatedExistence := '!' key
equality    := key '=' value
inequality  := key '!=' value
setIn       := key ' in (' value (',' value)* ')'
setNotIn    := key ' notin (' value (',' value)* ')'
```

## Security considerations

* Label/annotation values are never interpolated into SQL text — the selector parser builds a predicate tree and binds values as JDBC parameters, closing JSONB-based SQL injection
* Cursor is opaque and server-signed-free (plain base64 of `(name, id)`) — not a security boundary, just an implementation detail; no sensitive data is encoded in it

## Observability

* Log per list call: `{kind, project, environment, selector_terms, result_count, has_more}`
* Metric `forge_resource_list_requests_total{kind=...}`
* Metric `forge_resource_list_page_size` (histogram)

## Failure handling

* Malformed `labelSelector` syntax → `400 invalid_label_selector` with the offending term in `details`
* Invalid/expired cursor (fails base64 or references a name no longer present) → `400 invalid_cursor` — the client must restart the list from the beginning, it is not a `410` (that status is reserved for watch cursors in `20.05`)
* `limit` above `FORGE_LIST_MAX_PAGE_SIZE` → silently clamped (not an error), logged at `debug`
