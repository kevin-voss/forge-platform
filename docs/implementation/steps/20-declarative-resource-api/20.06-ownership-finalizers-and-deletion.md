## Goal

Make resource deletion safe for stateful platform capabilities by adding owner references, finalizer semantics, terminating state, and explicit delete-confirmation hooks. Deletes become reconciled workflows instead of immediate row removal.

## Why this step exists

Future resources such as `Database`, `Bucket`, `Volume`, `Certificate`, and `Backup` cannot be silently removed because a parent disappeared. Forge needs a general deletion protocol where controllers get a chance to drain traffic, snapshot data, revoke credentials, and remove external infrastructure before the API hides the resource.

## Dependencies

## Scope

* Owner reference validation: referenced owner exists in the same or wider scope.
* Finalizer mutation API for controllers.
* DELETE marks `metadata.deletionTimestamp`, sets phase `Terminating`, and emits `MODIFIED`, not immediate hard delete when finalizers exist.
* Hard deletion after all finalizers are removed.
* `requiresDeleteConfirmation` enforcement for stateful kinds.
* Conservative cascading: children are marked terminating only when the kind descriptor explicitly allows cascade.

## Explicitly out of scope

* Project-wide deletion UX and confirmation flows beyond the generic API primitive.
* Provider-specific cleanup logic; each owning controller implements its own finalizer action in later epics.

## Contracts

Finalizer patch:

```http
PATCH /v1/projects/invoice-platform/environments/production/applications/invoice-api/finalizers
```

```json
{ "add": ["deployment.forge.dev/finalizer"] }
```

Stateful delete:

```http
DELETE /v1/projects/invoice-platform/environments/production/databases/invoice-db
X-Forge-Delete-Confirmation: invoice-db
```

## Data model

No new table required. This step activates the `owner_refs`, `finalizers`, `deletion_timestamp`, and `deleted_at` fields created in `20.01`. If `deletion_timestamp IS NOT NULL` and finalizers are empty, the repository sets `deleted_at` and emits a terminal `DELETED` event.

## Observability

* Log finalizer add/remove with `{resource_id, finalizer, actor}`.
* Metric `forge_resource_terminating_total{kind}`.
* Event `resource.deletion_blocked` when a finalizer keeps a resource visible.

## Failure handling

* Missing delete confirmation for protected kind -> `409 delete_confirmation_required`.
* Finalizer removal by a non-owning controller -> `403 forbidden_finalizer`.
* Owner reference cycle -> `400 owner_reference_cycle`.

* DELETE with finalizers leaves the resource visible in `Terminating`.
* Removing the last finalizer performs the terminal delete and emits `DELETED`.
* Protected kinds require explicit name confirmation.
* Owner-reference cycles are rejected.
