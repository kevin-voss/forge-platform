## Goal

Create the end-to-end demo proving Forge accepts a portable declarative manifest, records generation changes, emits watch events, reconciles the deployment, and reaches `Ready` without breaking existing current-roadmap behavior.

## Scope

* `demos/20-declarative-resources`.
* Portable `Application` manifest using the future envelope.
* `forge apply`, `forge get`, and `forge watch` paths.
* Label listing and watch replay checks.
* Regression checks for legacy project/application/deployment endpoints.

## Demo flow

```text
Apply Application manifest
-> generation becomes 1
-> watch emits ADDED
-> reconciler creates Deployment
-> status becomes Deploying
-> Runtime reports ready
-> status observedGeneration catches up
-> status becomes Ready
-> apply an image update
-> generation increments
-> watch emits MODIFIED and STATUS_MODIFIED
```
