## Goal

Introduce the Kotlin types for the `apiVersion`/`kind`/`metadata`/`spec`/`status` envelope, an in-memory `KindRegistry` that later steps and later epics register kinds into, and the single generic `control.resources` storage table with ULID-prefixed ids. No HTTP route is added in this step — only an internal repository used directly by unit and integration tests.

## Why this step exists

Every later step in this epic (CRUD, concurrency, status, labels, watch, ownership, the compat facade) needs a shared envelope shape and a shared table to operate on. Building the types, the registry, and the schema first — and proving them with a real Postgres-backed repository before any route exists — keeps the storage contract honest and lets `20.02` focus purely on HTTP semantics instead of also inventing the data model under time pressure.

## Scope

* `ResourceMetadata` — id, name, organization, project?, environment?, generation, resourceVersion, labels, annotations, ownerRefs, finalizers, createdAt, updatedAt, deletionTimestamp?, deletedAt?
* `ResourceEnvelope` — apiVersion, kind, metadata, spec (`JsonObject`), status (`JsonObject`)
* `ResourceScope` enum — `Cluster`, `Project`, `Environment`
* `KindDescriptor` — kind, plural, scope, `parentKind: String?` (unused until `20.07`), schemaVersion, owningController, idPrefix, `requiresDeleteConfirmation: Boolean` (unused until `20.06`), `enforceScopeUniqueness: Boolean` (default `true`, unused until `20.07`)
* `KindRegistry` — thread-safe in-memory register/lookup by kind name and by plural
* `Ulid` — 128-bit ULID generator (48-bit ms timestamp + 80-bit randomness, Crockford Base32), no new third-party dependency
* `ResourceRepository` interface + `JdbcResourceRepository` implementation (insert, findById, findByScopeAndName) backed by `control.resources` — designed so `20.07` can add a second implementation backed by the legacy tables without changing the interface
* Migration creating `control.resources`, `control.resource_version_seq`, and scope-uniqueness indexes

## Explicitly out of scope

* Any HTTP route (`20.02`)
* PATCH/merge semantics, `resourceVersion` conflict handling (`20.02`)
* `/status` subresource, conditions, generation-bump rules (`20.03`)
* Label selectors, listing, pagination (`20.04`)

## Data model

```sql
CREATE SEQUENCE control.resource_version_seq;

CREATE TABLE control.resources (
    id                 TEXT PRIMARY KEY,
    kind               TEXT NOT NULL,
    api_version        TEXT NOT NULL DEFAULT 'forge.dev/v1',
    organization       TEXT NOT NULL,
    project            TEXT,
    environment        TEXT,
    name               TEXT NOT NULL,
    generation         BIGINT NOT NULL DEFAULT 1,
    resource_version   BIGINT NOT NULL,
    labels             JSONB NOT NULL DEFAULT '{}'::jsonb,
    annotations        JSONB NOT NULL DEFAULT '{}'::jsonb,
    spec               JSONB NOT NULL DEFAULT '{}'::jsonb,
    status             JSONB NOT NULL DEFAULT '{}'::jsonb,
    owner_refs         JSONB NOT NULL DEFAULT '[]'::jsonb,
    finalizers         JSONB NOT NULL DEFAULT '[]'::jsonb,
    created_at         TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at         TIMESTAMPTZ NOT NULL DEFAULT now(),
    deleted_at         TIMESTAMPTZ,
    CONSTRAINT resources_kind_not_blank CHECK (length(btrim(kind)) > 0),
    CONSTRAINT resources_name_not_blank CHECK (length(btrim(name)) > 0),
    CONSTRAINT resources_org_not_blank CHECK (length(btrim(organization)) > 0),
    CONSTRAINT resources_env_requires_project CHECK (environment IS NULL OR project IS NOT NULL)
);

-- Scope-aware uniqueness: NULL project/environment must not defeat the check
-- (plain UNIQUE treats NULL as distinct in Postgres), so each scope gets its
-- own partial index. Soft-deleted rows (deleted_at set) free the name.
CREATE UNIQUE INDEX resources_scope_unique_env ON control.resources
    (kind, organization, project, environment, name)
    WHERE deleted_at IS NULL AND environment IS NOT NULL;

CREATE UNIQUE INDEX resources_scope_unique_project ON control.resources
    (kind, organization, project, name)
    WHERE deleted_at IS NULL AND project IS NOT NULL AND environment IS NULL;

CREATE UNIQUE INDEX resources_scope_unique_cluster ON control.resources
    (kind, organization, name)
    WHERE deleted_at IS NULL AND project IS NULL;

CREATE INDEX resources_kind_scope_idx ON control.resources (kind, organization, project, environment);
```

`resource_version` is populated from `nextval('control.resource_version_seq')` at insert/update time in application code, not a column default — it must be assignable inside the same statement that also updates `updated_at`, and later steps (watch) need to read the value back.

## Failure handling

* Duplicate `(kind, organization, project, environment, name)` insert → Postgres unique-violation → repository throws the existing `forge.control.repo.RepositoryException.Conflict` (reused, not reinvented)
* `environment` set without `project` → CHECK constraint violation surfaces as `RepositoryException.ConstraintViolation`
