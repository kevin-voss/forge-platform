## Goal

Turn the inert evaluator from `24.01` into real workload autoscaling for CPU and memory metrics. The autoscaler computes desired replicas, applies stabilization and rate limits, and updates the target `Application` desired replica count through the resource API.

## Dependencies

* `24.01` - `ScalingPolicy`, evaluator, metric-source seam.
* `20.07` - `Application` available through declarative resource API.
* `25.01` is not required yet; this step scales replicas, not node capacity.

## Scope

* CPU and memory metric queries from Observe, with Runtime fallback for local mode.
* Utilization math: `ceil(currentReplicas * currentMetric / targetMetric)`.
* Min/max replica clamping.
* Scale-up and scale-down stabilization windows.
* `maxReplicasPerMinute` rate limiting.
* Status fields: currentReplicas, desiredReplicas, lastScaleTime, lastRecommendation.
* Actuation by patching `Application.spec.scaling.desiredReplicas`.

## Explicitly out of scope

* HTTP request/latency/error metrics.
* Queue-backed worker scaling.
* Node autoscaling.
* Scale-to-zero.

## Observability

* Metric `forge_autoscaler_recommendation_replicas{policy,target_kind,target_name}`.
* Metric `forge_autoscaler_scale_actions_total{direction,result}`.
* Span `autoscaler.actuate.application`.

## Failure handling

* Metric outage uses last safe desired replica count and sets `ScalingActive=Unknown`.
* Stale `Application` resource version retries with read-refresh and same operation id.
* Desired replicas are never set below `minReplicas` even during metric gaps.

## Implementation tasks

1. Implement CPU/memory queries in `ObserveSource` and local fallback in `RuntimeSource`.
2. Implement utilization replica math with clamp, stabilization, and rate limits.
3. Patch target `Application` desired replicas through the resource API.
4. Record actuation status and conditions.
5. Add unit tests for scaling math and integration tests with fake metric sequences.

## Acceptance criteria

* CPU/memory policies scale an `Application` up and down within min/max bounds.
* Stabilization windows prevent rapid downscaling.
* Rate limits cap replica changes per minute.
* Metric outages do not reduce availability.
