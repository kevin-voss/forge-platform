## Goal

Add time-based scaling schedules, temporary manual overrides, deployment-freeze awareness, and explicit metric-outage fallback behavior so autoscaling remains predictable during business cycles and incidents.

## Scope

* `ScalingPolicy.spec.schedules[]` with cron, timezone, min/max override, and optional end time.
* Manual override subresource with TTL and reason.
* Deployment freeze guard: no scale-down during active rollout or freeze window.
* Metric outage fallback: hold, floor, or manual fixed replicas.
* Audit entries for every override and schedule activation.

## Explicitly out of scope

* Calendar integrations.
* Cost-budget enforcement.
* Node autoscaling.

## Observability

* Metric `forge_autoscaler_schedule_active{policy}`.
* Metric `forge_autoscaler_manual_override_active{policy}`.
* Event `autoscaling.override.created`.

## Failure handling

* Invalid cron/timezone rejected at admission.
* Expired override automatically clears and emits an event.
* Conflicting schedules choose the highest `minReplicas` and lowest `maxReplicas` that still permit `min <= max`; otherwise policy condition `ScheduleConflict=True`.

## Implementation tasks

1. Add cron parsing and timezone validation.
2. Add manual override API with TTL and audit.
3. Merge metric recommendation, schedules, overrides, deployment freezes, and fallback in deterministic precedence order.
4. Add tests for overlapping schedules and expired overrides.
5. Document operations runbook for pausing or forcing autoscaling.

## Acceptance criteria

* Scheduled min/max changes take effect at the correct time.
* Manual overrides supersede metrics until TTL expires.
* Active deployment/freeze windows block scale-down but allow safe scale-up.
* Metric outage behavior is explicit in status.
