## Goal

Safely remove underutilized runtime nodes by cordoning, draining, respecting disruption budgets and stateful placement rules, then asking Forge Infrastructure to delete empty nodes.

## Dependencies

* `24.06` - node autoscaler and NodePool status.
* `25.04` - disruption budget guard.
* `25.05` - stateful placement constraints for protected workloads.
* `23.03` - node drain and delete lifecycle.

## Scope

* Underutilized-node detection over configurable windows.
* Cordon: scheduler stops placing new workloads on candidate node.
* Drain: move eligible workloads elsewhere before delete.
* Safeguards: min nodes, minimum availability, no active deployment, stateful primary protection, cooldowns, max deletes per window.
* Delete via Infrastructure only after node is empty.
* Status/audit for declined scale-down decisions.

## Explicitly out of scope

* Spot/preemptible interruption handling.
* Cross-region rebalancing.
* Cost optimizer recommendations beyond safe removal.

## Observability

* Metric `forge_node_autoscaler_scale_down_candidates_total`.
* Metric `forge_node_autoscaler_drains_total{result}`.
* Event `node.drain.started` / `node.drain.completed`.
* Span `autoscaler.node.scale_down`.

## Failure handling

* Drain blocked by disruption budget -> node remains cordoned or is uncordoned based on configured retry policy.
* Replacement placement unavailable -> scale-down canceled and node uncordoned.
* Infrastructure delete failure -> node enters `DeleteBlocked`; retry with same operation id.

## Implementation tasks

1. Implement underutilization scoring and candidate selection.
2. Add cordon/drain orchestration.
3. Wire disruption budget and stateful placement guards.
4. Delete empty nodes through Infrastructure.
5. Add restart recovery for in-progress drains.
6. Add tests for blocked, canceled, and successful scale-down.

## Acceptance criteria

* Underutilized nodes are drained and removed without dropping below availability budgets.
* Stateful primary workloads are never scaled down automatically.
* Scale-down is canceled safely when replacement placement cannot be found.
* Provider deletion is idempotent.
