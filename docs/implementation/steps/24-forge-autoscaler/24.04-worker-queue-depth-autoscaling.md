## Goal

Scale worker workloads from durable queue signals: queue depth, oldest message age, processing duration, consumer lag, retry rate, and dead-letter pressure.

## Dependencies

* `24.01` - metric-source seam and `ScalingPolicy`.
* Epic `28-forge-queue` for the long-term product queue. Until then, this step can use Forge Events/NATS metrics as the local implementation path.

## Scope

* `Worker` target kind with `ScalingPolicy.spec.targetRef.kind=Worker`.
* Queue metric adapter reading Forge Events/NATS stream metadata in local mode.
* Desired replicas based on backlog per worker and oldest-message SLO.
* Cooldown and max-replica safeguards identical to application autoscaling.
* Status explaining backlog, oldest age, and recommended worker count.

## Explicitly out of scope

* Implementing Forge Queue itself.
* Job lease/visibility-timeout semantics.
* Autoscaling agents or model servers.

## Observability

* Metric `forge_autoscaler_queue_backlog{queue}`.
* Metric `forge_autoscaler_worker_desired_replicas{worker}`.
* Span `autoscaler.metrics.queue`.

## Failure handling

* Queue metric unavailable -> keep last desired worker count unless manual override says otherwise.
* Retry-rate spike -> scale-up is allowed but scale-down is blocked until retry rate recovers.
* Empty queue for longer than stabilization window -> scale down no lower than `minReplicas`.

## Implementation tasks

1. Define the `Worker` target contract and resource references.
2. Implement queue metric adapter for Forge Events/NATS.
3. Add backlog and oldest-age replica math.
4. Patch `Worker` desired replicas through the resource API.
5. Add tests for burst, drain, retry spike, and metric outage scenarios.

## Acceptance criteria

* A backlog increase scales workers up within policy limits.
* Once the queue drains, workers scale down after stabilization.
* Retry pressure blocks unsafe downscaling.
* The implementation remains portable and does not depend on provider queues.
