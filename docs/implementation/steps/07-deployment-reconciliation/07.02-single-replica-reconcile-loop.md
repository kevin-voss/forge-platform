## Goal

Make the reconciliation controller **act**: execute the plan computed in 07.01 to converge actual replicas to desired for a single service version. Missing replicas are created via Runtime, surplus replicas are stopped, and a replica that has crashed/exited is recreated — all idempotently, so repeated ticks and controller restarts never produce duplicate containers.

## Why this step exists

07.01 produced an inert plan. Convergence is the core value of the epic and the foundation for rolling updates (07.03) and rollback (07.04). Doing single-version convergence first isolates the "create/stop/restart" mechanics and idempotency guarantees from the more complex multi-version rollout logic.

## Dependencies

* Step `07.01` — desired/actual model, `computePlan`, controller loop, `RuntimeClient`
* Epic `04-forge-runtime` — workload create/start (`04.03`), health/status (`04.04`), stop/delete without duplicates (`04.06`), Control integration (`04.07`)

## Scope

* Execute `StartReplica` actions: call Runtime to create+start a workload with a deterministic name/label derived from `{deployment_id, replica_index}`
* Execute `StopReplica` actions: stop the highest-index surplus replica
* Detect crashed/exited replicas (status `failed`/`stopped` while still desired) and recreate them
* Idempotency: before starting, check Runtime for an existing workload with the deterministic name/label; never create a duplicate
* Update `actual_json` in `reconcile_status` after each execution pass
* Bounded actions per tick (respect a max to avoid thundering herd)

## Explicitly out of scope

* Rolling update between two image versions / traffic shifting (07.03)
* Automatic rollback on unhealthy rollout (07.04)
* Durable transition history beyond current snapshot (07.05)
* Choosing which node runs a replica (epic 08 — all replicas go to the single local Runtime here)

## Architecture

```text
tick:
  desired, actual ← load
  plan ← computePlan(desired, actual)
  for action in plan (bounded):
    StartReplica(i) → ensureWorkload(deployment, i)   # idempotent create
    StopReplica(i)  → RuntimeClient.stop(name(deployment,i))
  actual' ← RuntimeClient.observe(deployment)
  upsert reconcile_status(actual')
```

Deterministic workload identity:

```text
name  = forge-<service_slug>-<deployment_short>-<replica_index>
labels: forge.deployment=<id>, forge.service=<id>, forge.replica=<index>, forge.image=<ref>
```

## Security considerations

* Deterministic names prevent accidental adoption of unrelated containers (label match is required before adopt)
* Controller only manages workloads carrying `forge.deployment` labels it owns
* No secrets handled here (secret injection is epic 10)

## Observability

* Log per executed action: `{deployment_id, action, replica_index, result, duration_ms}`
* Metrics: `forge_reconcile_actions_total{action=start|stop|recreate}`, `forge_replicas_ready` gauge
* Trace child spans `reconcile.start_replica` / `reconcile.stop_replica`

## Failure handling

* Runtime create fails → log, leave desired>actual, retry next tick (self-healing)
* Duplicate-name conflict from Runtime → treat as success (idempotent; already exists)
* Stop fails → retry next tick; never leave two competing definitions
* Controller restart mid-execution → next tick re-observes actual and recomputes; no duplicates because create is guarded by label lookup

## Manual verification

```bash
make dev
DPL=$(curl -s -X POST localhost:4001/v1/deployments \
  -d '{"service_id":"svc_demo","image":"registry.local/demo:v1","desired_replicas":2}' \
  -H 'content-type: application/json' | jq -r .id)

sleep 6
docker ps --filter "label=forge.deployment=$DPL" --format '{{.Names}}'   # expect 2 containers

# Idempotency: another tick creates nothing new
sleep 3
test "$(docker ps --filter "label=forge.deployment=$DPL" -q | wc -l | tr -d ' ')" = "2"

# Self-heal: kill one, controller recreates
docker kill "$(docker ps --filter "label=forge.deployment=$DPL" -q | head -1)"
sleep 6
test "$(docker ps --filter "label=forge.deployment=$DPL" -q | wc -l | tr -d ' ')" = "2" && echo OK-recreated
```

## Acceptance criteria

* Desired and actual replica counts converge for a single version
* A crashed/killed replica is automatically recreated
* Surplus replicas are stopped when desired count decreases
* Repeated ticks and controller restart never create duplicate containers (idempotent by label)
* Runtime failures are retried on subsequent ticks without crashing the controller
