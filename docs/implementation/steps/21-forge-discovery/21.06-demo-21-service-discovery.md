## Goal

Prove Forge Discovery provides stable, health-aware service discovery for local multi-runtime workloads and that Gateway can source routes from Discovery behind the reversible `FORGE_ROUTE_SOURCE=discovery` flag.

## Scope

* Demo with at least three services and multiple endpoints.
* Internal `.svc.forge` DNS lookups.
* Readiness-aware endpoint selection.
* Endpoint lease expiry failure path.
* Gateway route-source flip and rollback.

## Demo flow

```text
Runtime starts users-api replica
-> Runtime registers endpoint lease
-> health probe succeeds
-> Discovery marks endpoint Ready
-> internal DNS resolves users.production.shop.svc.forge
-> Gateway routes through Discovery
-> one Runtime heartbeat expires
-> endpoint becomes Unready
-> Gateway stops routing to it
-> replacement endpoint registers and becomes Ready
```
