## Goal

Runtime reports each replica's private endpoint to Discovery right after it passes its first readiness probe, renews a TTL lease on its existing probe cadence, and Discovery expires unrenewed leases to `Unready`. Independently, Discovery watches the epic-08 `Node` resource's `Reachable` condition and, on transition to unreachable, marks every `Endpoint` on that node `Unready` in a single transaction. Re-registration after a Runtime restart is idempotent — no duplicate endpoints.

## Why this step exists

`21.01` gave Discovery a place to store endpoints but nothing populates or expires them. This step wires the two failure-detection paths the epic's success demo depends on: a per-endpoint lease (catches a single replica dying without its node dying) and a node-level fast path (catches the whole node going dark without waiting out N individual lease timers).

## Dependencies

* `21.01` — service skeleton, `discovery.services`/`discovery.endpoints` tables, kind registration

## Scope

* `endpoints` table gains lease columns (`ready`, `lease_seconds`, `expires_at`, `unready_reason`)
* `POST /v1/projects/{project}/environments/{environment}/services/{service}/endpoints` — register (upsert by replica id)
* `POST /v1/projects/{project}/environments/{environment}/endpoints/{id}/renew` — renew lease + report current readiness
* `DELETE /v1/projects/{project}/environments/{environment}/endpoints/{id}` — explicit deregister on graceful stop
* Background sweeper: expires unrenewed leases → `Unready`; reaps long-`Unready` endpoints after a grace period
* Node-watch: subscribe to Control's generic watch for the `Node` kind (epic 20 conventions), react to `Reachable=False`
* Runtime changes: one outbound call after first readiness pass, one renew call per probe tick, one deregister call on graceful stop
* Async mirror: accepted writes are projected into Control's generic `Endpoint`/`Service` resources (best-effort, retried)

## Explicitly out of scope

* Readiness-filtered bulk selection API / SSE watch for consumers (`21.03`)
* DNS resolution (`21.04`)
* Gateway consuming any of this (`21.05`)

## Architecture

```text
forge-runtime (per replica)
  ├─ on first readiness pass ─▶ POST .../endpoints           (register)
  ├─ on every probe tick      ─▶ POST .../endpoints/{id}/renew (renew + ready flag)
  └─ on graceful stop         ─▶ DELETE .../endpoints/{id}     (deregister)

forge-discovery
  ├─ sweeper (every FORGE_DISCOVERY_SWEEP_INTERVAL_SECONDS)
  │     expires_at < now() AND phase != 'Unready'  → phase='Unready', reason='LeaseExpired'
  │     phase='Unready' AND updated_at < now()-reap → DELETE (garbage collect)
  ├─ node-watch (GET /v1/watch/nodes?since=... on Control)
  │     Reachable→False  → UPDATE endpoints SET phase='Unready', reason='NodeUnreachable'
  │                         WHERE node_id = $1 AND phase != 'Unready'   (one transaction)
  └─ mirror worker → PUT/PATCH Control generic Endpoint/Service resources (best-effort, retried)
```

## Components

| Component | Responsibility |
|---|---|
| `endpoints.Register` | Idempotent upsert of an `Endpoint` row keyed by replica id |
| `endpoints.Renew` | Resets `expires_at`, updates `ready`, flips `phase` Ready/Unready |
| `endpoints.Deregister` | Immediate removal on graceful stop |
| `sweeper.Run` | Periodic expiry + reap loop |
| `nodewatch.Subscribe` | Watches `Node` resources; applies node-level unready transactionally |
| `mirror.Worker` | Async, retried projection of local state into Control's generic API |
| Runtime `discoveryclient` | New Rust module issuing register/renew/deregister calls |

## Data model

```sql
ALTER TABLE discovery.endpoints
  ADD COLUMN ready          BOOLEAN     NOT NULL DEFAULT false,
  ADD COLUMN lease_seconds  INT         NOT NULL DEFAULT 20,
  ADD COLUMN expires_at     TIMESTAMPTZ NOT NULL DEFAULT now() + interval '20 seconds',
  ADD COLUMN unready_reason TEXT;

CREATE INDEX idx_endpoints_expiry ON discovery.endpoints (expires_at) WHERE phase <> 'Unready';
CREATE INDEX idx_endpoints_node_phase ON discovery.endpoints (node_id, phase);
```

Node-unreachable transactional update (one statement, one transaction):

```sql
UPDATE discovery.endpoints
   SET phase = 'Unready', ready = false, unready_reason = 'NodeUnreachable', updated_at = now()
 WHERE node_id = $1 AND phase <> 'Unready';
```

## Security considerations

* Registration/renew/deregister trust the calling node's identity (dev-mode bypass until epic 09; production posture requires the workload/node mTLS identity epic 09 issues to match `spec.node`)
* Node-watch and mirror-worker calls to Control carry an idempotency/operation id per the platform's cross-cutting rule so retried mirrors never double-apply
* A replica cannot register an endpoint for a service it does not belong to (validated against the deployment's declared service name)

## Observability

* Log per registration: `{event:"discovery.endpoint.registered", id, service, node, address}`
* Log per expiry: `{event:"discovery.endpoint.unready", id, reason: "LeaseExpired"|"NodeUnreachable"}`
* Log per node-unready batch: `{event:"discovery.node.unreachable", node_id, endpoints_affected}`
* Metrics: `forge_discovery_endpoints_total{phase=...}`, `forge_discovery_lease_expirations_total`, `forge_discovery_node_unready_batches_total`
* Trace spans: `discovery.endpoint.register`, `discovery.endpoint.renew`, `discovery.node.unready_batch`

## Failure handling

* Runtime crashes without deregistering → lease expires within `lease_seconds` + one sweep interval → `Unready` → reaped after `FORGE_DISCOVERY_REAP_AFTER_SECONDS`
* Runtime restarts → re-registration uses the same replica id → idempotent upsert, no duplicate `Endpoint` rows
* Control unreachable → mirror worker retries with backoff; registration/renew/deregister against Discovery's own store keep working (local store is authoritative for serving)
* Node-watch stream drops → full resync every `FORGE_DISCOVERY_NODE_WATCH_RESYNC_SECONDS` catches any missed transition

## Acceptance criteria

* Registration is idempotent per replica id; renew updates readiness and resets the lease
* Unrenewed leases expire to `Unready` within `lease_seconds` + one sweep interval
* A `Node` transitioning to `Reachable=False` marks all of its endpoints `Unready` in one transaction, independent of individual lease timers
* Runtime restart never produces duplicate `Endpoint` rows for the same replica
* Accepted writes are mirrored into Control's generic API (eventually consistent; retried on Control outage)
