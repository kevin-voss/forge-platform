## Goal

Harden the deployed product with platform foundations: authenticate/authorize via Identity, source config/credentials from Secrets (no hardcoded values), emit telemetry to Observe (logs/metrics/traces), use Storage for artifacts/uploads, and attach a managed Postgres so the product persists data via an injected `DATABASE_URL`.

## Why this step exists

`specs.md` Step 19 scenario steps 5–6 (secrets, telemetry) plus the acceptance criteria (identity/permission, secret injection, telemetry, isolated DB) require these foundations wired into the real product.

## Dependencies

* `19.02` (deployed, reachable product)
* Identity `09.08`, Secrets `10.07`, Observe `12.07`, Storage `13.07`, Managed DB `18.06`

## Scope

* Identity: product/API access requires auth; a viewer vs deployer role difference demonstrated
* Secrets: product config + DB URL delivered via Secrets injection; masked in logs
* Observe: product services instrumented; one distributed trace across services visible
* Storage: product stores an artifact (e.g. an uploaded log bundle) in a project bucket
* Managed DB: `forge database create/attach` → product persists incidents in isolated Postgres

## Explicitly out of scope

* Models/Agents/Memory (19.04)
* Failure injection + Workflows (19.05)
* Acceptance suite (19.06)

## Architecture

```text
Identity → authz on product + platform mutations
Secrets  → product config + DATABASE_URL (injected at deploy)
Observe  → traces/logs/metrics across the five services
Storage  → project bucket for artifacts
Managed DB → isolated Postgres attached to api-go/admin-kotlin
```

## Security considerations

* `FORGE_AUTH_MODE=dev` bypass removed for the product (enforced)
* No plaintext secrets in config/logs (masking verified)
* DB isolated from Control DB; least-privilege role
* Role-based access demonstrated (viewer cannot deploy)

## Observability

* Distributed trace across ≥3 services for one product request
* Product metrics + structured logs queryable by project/deployment

## Failure handling

* Missing secret/DB URL at deploy → deploy fails clearly (no silent misconfig)
* Unauthorized request → 401/403
* Storage/Observe unavailable → product degrades gracefully where documented
