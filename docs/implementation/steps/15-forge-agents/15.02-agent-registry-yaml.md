# Step 15.02: Agent registry + YAML definitions

## Status

Not started

## Epic

`15-forge-agents` → [`../../epics/15-forge-agents.md`](../../epics/15-forge-agents.md)

## Goal

Load agent definitions from YAML into a validated registry and expose `GET /v1/agents` and `GET /v1/agents/{name}`, capturing model selection, declared tools, permissions, and limits per the `specs.md` agent schema.

## Why this step exists

Agents must be declaratively defined and validated before they can run. The registry is the source of truth the run engine (15.04) and permission checks (15.03) consult.

## Dependencies

* `15.01` (skeleton)

## Scope

* Agent YAML schema + loader with validation
* Registry keyed by agent `name`
* `GET /v1/agents`, `GET /v1/agents/{name}`
* Validation: referenced tools/permissions well-formed (existence of tools enforced in 15.03), limits within bounds
* Load definitions from `services/forge-agents/agents/*.yaml`

## Explicitly out of scope

* Tool registry + permission enforcement (15.03)
* Run engine (15.04)
* Seed agent content (15.07 — here only a minimal fixture agent)

## Architecture

```text
agents/*.yaml → AgentLoader → validate → { name → AgentDefinition }
GET /v1/agents → list; GET /v1/agents/{name} → detail | 404
```

## Components

| Component | Purpose |
|---|---|
| `AgentDefinition` model | Typed schema (pydantic) |
| `AgentLoader` | Parse + validate YAML files |
| `agents` routes | List/get |

## Contracts

* `GET /v1/agents` → `200 { "agents":[{ "name","model","tools","permissions","limits" }] }`
* `GET /v1/agents/{name}` → `200 {agent}` | `404 agent_not_found`
* Extend OpenAPI

## Data model

Agent definition (from `specs.md` Step 15):

```yaml
name: deployment-investigator
model: local-general
tools: [deployment.read, logs.search, metrics.query, runtime.restart]
permissions: [project:read, deployment:read]
limits:
  max_steps: 10
  timeout_seconds: 120
```

## Configuration

```text
FORGE_AGENTS_DEFS_DIR   # default services/forge-agents/agents
```

## Security considerations

* Definitions validated; unknown/malformed fields rejected
* `max_steps`/`timeout_seconds` bounded (reject absurd values)
* No secrets in definitions

## Observability

* Log loaded agents (count, names)
* Metric: `agents_registry_size`

## Failure handling

* Malformed YAML → startup failure with file + reason
* Duplicate agent name → startup failure
* Unknown agent name over API → `404`
