## Goal

Stand up the `forge-agents` Python (FastAPI) service with health endpoints, structured logging, graceful shutdown, a Docker image, and Compose wiring on host port `4301` — no agent/tool logic yet.

## Why this step exists

The agent runtime needs a running, contract-compliant service before registries, tools, and the run engine can be layered on. Isolating the scaffold keeps later steps focused on agent semantics.

## Dependencies

* Epics `00`, `01` conventions (health, logs, `PORT`, SIGTERM)

## Scope

* FastAPI app under `services/forge-agents`
* `GET /health/live`, `GET /health/ready`, `GET /` identity JSON
* Config via env (`PORT`, `FORGE_LOG_LEVEL`, `FORGE_MODELS_URL`)
* Structured JSON logging + request IDs
* Dockerfile + Compose service on `4301`
* Makefile targets; port reservation confirmed
