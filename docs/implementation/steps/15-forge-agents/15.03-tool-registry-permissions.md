# Step 15.03: Tool registry + per-call permission checks

## Status

Not started

## Epic

`15-forge-agents` → [`../../epics/15-forge-agents.md`](../../epics/15-forge-agents.md)

## Goal

Implement a tool registry with typed input/output schemas and enforce that an agent may only invoke tools it declares, that each tool call passes a permission check against the run's identity/project scope before execution, and that unknown/hallucinated tools are rejected.

## Why this step exists

The core safety property of the epic is that agents call only registered tools and permissions are checked before every call (`specs.md` Step 15 acceptance). Establishing this before the run engine ensures the loop cannot bypass guardrails.

## Dependencies

* `15.02` (agent registry)

## Scope

* `Tool` interface: name, JSON input/output schema, `destructive` flag, required permission(s), `execute()`
* Tool registry with schema validation of arguments
* Permission model + checker: `has_permission(scope, required)`
* Enforcement points: tool must be (a) declared by the agent, (b) registered, (c) permitted — else rejected
* Hallucinated-tool rejection (name not in registry)
* Fake/mock tools for testing (real platform tools in 15.05)

## Explicitly out of scope

* Real platform tool implementations (15.05)
* Approval gating for destructive tools (15.06)
* Run loop (15.04)

## Architecture

```text
agent.tools ∩ registry → allowed set
call(tool, args, scope):
  assert tool in agent.tools           # declared
  assert tool in registry              # exists (else hallucination)
  validate(args, tool.input_schema)    # schema
  assert has_permission(scope, tool.required_permissions)  # authz
  return tool.execute(args)            # (approval added in 15.06)
```

## Components

| Component | Purpose |
|---|---|
| `Tool` base | Contract for all tools |
| `ToolRegistry` | Name → tool; schema lookup |
| `PermissionChecker` | Match required perms to scope |
| `ToolInvoker` | Runs the enforcement pipeline |
| Fake tools | `echo`, `fail`, `deployment.read` (stub) for tests |

## Contracts

* `GET /v1/tools` → `200 { "tools":[{ "name","input_schema","output_schema","destructive","required_permissions" }] }`
* Internal invocation contract (used by run engine): rejection reasons `unknown_tool`, `not_declared`, `permission_denied`, `invalid_arguments`
* Extend OpenAPI (tool listing)

## Data model

No persistence. Permission strings mirror agent `permissions` (e.g. `deployment:read`, `runtime:restart`).

## Configuration

```text
FORGE_AGENTS_TOOLS_MODE   # fake|live (default fake for CI)
```

## Security considerations

* Deny-by-default: any check failure aborts the call and is audited
* Argument schema validation prevents malformed tool inputs
* Distinguish `unknown_tool` (hallucination) from `not_declared` (agent overreach) in logs/audit

## Observability

* Log every tool-call decision with reason (allow/deny + which check)
* Metric: `agent_tool_calls_total{tool,decision}`, `agent_tool_denied_total{reason}`

## Failure handling

* Unknown tool → reject `unknown_tool`
* Tool not declared by agent → reject `not_declared`
* Missing permission → reject `permission_denied`
* Bad arguments → reject `invalid_arguments`
