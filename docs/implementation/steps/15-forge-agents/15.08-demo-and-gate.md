# Step 15.08: Demo `15-agent-runtime` + gate

## Status

Not started

## Epic

`15-forge-agents` → [`../../epics/15-forge-agents.md`](../../epics/15-forge-agents.md)

## Goal

Deliver `demos/15-agent-runtime` and a one-command acceptance suite: a deliberately failing deployment is created, the deployment-investigator agent inspects status/logs/readiness, produces a diagnosis and recommendation, and does NOT restart anything without human approval — the epic acceptance gate.

## Why this step exists

The epic is done only when the full safety story is demonstrated end-to-end: registered-tools-only, permission checks, enforced limits, auditability, and no unauthorized destructive action (`specs.md` Step 15 acceptance).

## Dependencies

* `15.07` (seed agents + CLI)
* Backing services in fake mode (deterministic CI) or live locally

## Scope

* `demos/15-agent-runtime/` Compose wiring `forge-agents` + `forge-models` (fake) + fixtures for a failing deployment
* Acceptance script that runs the investigator and asserts: diagnosis produced, recommendation includes restart, restart NOT executed (awaiting approval), hallucinated tool rejected
* Demo README
* Wire `make demo DEMO=15`

## Explicitly out of scope

* New agent features (15.01–15.07)
* Workflow orchestration (epic 16)
* Memory retrieval (epic 17)

## Demo scenario

```text
1. bring up forge-agents + forge-models(fake) with failing-deployment fixtures
2. run deployment-investigator
3. agent reads deployment status, logs, readiness failure (registered tools only)
4. agent produces a diagnosis + recommends restart
5. restart is requested → run is awaiting_approval; nothing is restarted
6. a hallucinated tool call (negative case) is rejected
7. run history is complete and auditable
```

## Acceptance criteria

* `make demo DEMO=15` exits `0`
* The investigator diagnoses the failing deployment using only registered, permitted tools
* The recommended restart is NOT executed without approval (run is `awaiting_approval`)
* A hallucinated/unregistered tool call is rejected
* Execution limits are respected and the run history is auditable
* Demo README documents the flow; deterministic in CI
* Epic 15 marked Complete in `progress.md` only after this gate passes
