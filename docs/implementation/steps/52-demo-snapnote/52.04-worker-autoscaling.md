## Goal

Scale the worker by queue depth: a burst of uploads grows worker replicas within bounds, the
backlog drains, then replicas scale back down; scale-down is blocked while retries are in flight.
Optionally, if replicas can't be placed, Infrastructure provisions a Docker node.

## Dependencies

* `52.03`; Autoscaler (epic 24, queueDepth); optional Infrastructure (epic 23).

## Scope

* `ScalingPolicy { type: queueDepth, queue: snapnote-attachments, targetPerReplica }` on the worker
  (`minReplicas`/`maxReplicas`).
* A burst generator (harness or `run.sh`) enqueues N attachments.
* Assert replica direction + bounds; scale-down safety while messages retrying.
* Optional: push worker demand beyond capacity → Docker node provisioned then drained.

## Explicitly out of scope
