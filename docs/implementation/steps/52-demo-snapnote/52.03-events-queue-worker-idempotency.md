## Goal

Process attachments asynchronously: publish `attachment.uploaded` to a durable queue and consume it
in a background worker that generates a thumbnail and flips status to `ready` — exactly once.

## Dependencies

* `52.02`; Events (epic 11).

## Scope

* `dependencies.queue { type: durable, name: snapnote-attachments }`; API publishes on upload.
* `snapnote-worker` (Worker resource) consumes with ack + idempotency key (`attachment_id`), fetches
  the object, produces a thumbnail, PUTs it back, updates `status=ready, thumbnail_key`.
* Idempotency: reprocessing the same message does not create duplicate thumbnails.

## Explicitly out of scope

* Autoscaling (52.04) — a single worker replica here.
