## Goal

When desired replicas exceed cluster capacity, Infrastructure provisions a Docker node so the extra
replicas can run; when demand falls, the idle node drains and is deleted.

## Dependencies

* `55.02`; Infrastructure (epic 23, docker provider); Autoscaler node path (epic 24).

## Scope

* Operator-owned `NodePool { provider: docker, minNodes, maxNodes }`.
* Push load high enough (or set replica resources) that placement fails → node scale-up → extra
  replicas Running; then scale-down + drain after load stops.
* Assert node-count direction + bounds; scale-down respects drain safeguards.
