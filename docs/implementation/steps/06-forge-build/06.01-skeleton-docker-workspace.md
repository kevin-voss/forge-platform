## Goal

Stand up the `forge-build` Go service as a health-checked container on host port `4103` that can reach Docker Engine and manage an isolated workspace directory for transient clones, with env-based config, structured logs, graceful shutdown, and Compose wiring — but no build logic yet.

## Why this step exists

The build steps (clone, docker build, push) need a bootable service with Docker access and a place to check out source. Establishing the skeleton, Docker connectivity, and workspace lifecycle first isolates plumbing from build logic.

## Dependencies

* Prior steps in this epic: none (first step of epic 06)
* Cross-epic prerequisites: epic `00` (Docker Engine, local registry `localhost:5000`, Compose, ports); epic `01` health/log conventions

## Scope

* `services/forge-build/` Go service that builds and runs
* `GET /health/live` and `GET /health/ready` (ready requires Docker reachable)
* Docker Engine client wired to the socket
* Workspace manager: create/cleanup isolated per-build directories under a mounted volume
* Env config; structured JSON logs; graceful shutdown; `Dockerfile`, `Makefile`, Compose entry; port `4103` reserved

## Explicitly out of scope

* `forge.yaml` parsing / build API contract (06.02)
* Clone/checkout + docker build (06.03)
* Tag/push (06.04)
* Status/failure model (06.05)
* Control integration (06.06)

## Architecture

```text
Compose service `forge-build` (host 4103 → PORT 8080)
  mounts /var/run/docker.sock + workspace volume
        │
        ▼
Go server
├── /health/live   → 200
├── /health/ready  → 200 iff docker reachable
└── workspace mgr  → /workspace/<buildId>/ (created + cleaned)
```

## Components

| Component | Purpose |
|---|---|
| `main.go` | Boot server, load config, register health |
| `internal/config` | Env → typed config |
| `internal/docker` | Docker client factory + ping |
| `internal/workspace` | Isolated dir create/cleanup |
| health handlers | live/ready (ready pings Docker) |
| Compose entry | Service + socket + workspace volume + port |

## Contracts

Runtime-contract health surface: `/health/live` → `200`; `/health/ready` → `200` when Docker is reachable, else `503`. No build API yet.

## Data model

None persisted (build metadata store arrives with the job model in 06.03/06.05). Workspace dirs are transient on the volume.

## Configuration

```text
PORT=8080
FORGE_SERVICE_NAME=forge-build
FORGE_SERVICE_VERSION=0.1.0
FORGE_LOG_LEVEL=info
FORGE_ENV=development
FORGE_AUTH_MODE=dev
DOCKER_HOST=unix:///var/run/docker.sock
FORGE_BUILD_WORKSPACE_DIR=/workspace
FORGE_SHUTDOWN_GRACE_SECONDS=10
```

Host publish `4103:8080`.

## Security considerations

* Docker socket mount is a dev-only privileged convenience (like Runtime); flagged for hardening.
* Workspace dirs are created with restrictive permissions and always cleaned to avoid leaking source.
* No secrets logged.

## Observability

* Structured JSON logs (`timestamp`, `level`, `service`, `message`).
* Startup logs Docker Engine version + workspace dir.

## Failure handling

* Docker unreachable → readiness `503` (bounded retry).
* Workspace dir not writable → fail startup with a clear error.
* Invalid `PORT` → exit non-zero; `SIGTERM` → graceful shutdown.
