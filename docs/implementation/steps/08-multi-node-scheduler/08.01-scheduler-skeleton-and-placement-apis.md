# Step 08.01: Scheduler module/service skeleton + placement APIs

## Status

Not started

## Epic

`08-multi-node-scheduler` → [`../../epics/08-multi-node-scheduler.md`](../../epics/08-multi-node-scheduler.md)

## Goal

Create the scheduler as a self-contained module inside Forge Control with a stable placement interface (`Scheduler.place(PlacementRequest) → PlacementDecision`) and a read/write placement API, backed by a trivial single-node strategy for now. Establish the extract seam so the scheduler can later move to a standalone service on port `4108` without changing its contract.

## Why this step exists

Later steps add node fleet tracking, real strategies, affinity, and rescheduling. They all depend on a defined placement boundary and persistence. Building the seam + inert strategy first keeps the interface honest and lets the reconciler start delegating placement immediately (even if the answer is always "the one node").

## Dependencies

* Epic `07-deployment-reconciliation` — reconcile loop that will request placements (`07.02`)
* Epic `02-forge-control` — Postgres + module structure
* No prior steps in this epic

## Scope

* `Scheduler` interface + `SingleNodeScheduler` default implementation (returns the only registered node)
* `PlacementRequest` (service, replica index, resource requirements, affinity hints) and `PlacementDecision` (node id, reason)
* Persist placements: which node each replica is assigned to
* Placement APIs: `POST /v1/placements` (compute+persist a placement) and `GET /v1/placements?deployment=...`
* Extract-seam documentation: the module exposes only the interface; no Control internals leak in
* OpenAPI additions for placement endpoints

## Explicitly out of scope

* Multi-node registration/heartbeat/resources (08.02)
* First-fit / least-allocated strategies (08.03)
* Anti-affinity + pending queue (08.04)
* Reschedule on node loss (08.05)
* Standalone binary / port 4108 (deferred; seam only)

## Architecture

```text
services/forge-control/
└── scheduler/                      ← module with extract seam
    ├── Scheduler.kt (interface)    place(req): decision
    ├── SingleNodeScheduler.kt      trivial impl (only node)
    ├── model/PlacementRequest.kt
    ├── model/PlacementDecision.kt
    ├── PlacementStore.kt           persist replica→node
    └── api/PlacementRoute.kt       POST/GET /v1/placements
reconcile/Reconciler → Scheduler.place(...)   (07 delegates here)
```

## Components

| Component | Responsibility |
|---|---|
| `Scheduler` (interface) | `place(request): decision` — the seam |
| `SingleNodeScheduler` | Default impl returning the sole node |
| `PlacementRequest` / `PlacementDecision` | Placement I/O types |
| `PlacementStore` | Persist replica→node assignment |
| `PlacementRoute` | HTTP surface for placement |

## Contracts

* `POST /v1/placements` request:

```json
{ "deployment_id": "dpl_1", "replica_index": 0,
  "requirements": { "slots": 1 }, "anti_affinity": "soft" }
```

* response:

```json
{ "placement_id": "plc_1", "deployment_id": "dpl_1", "replica_index": 0,
  "node_id": "node-local", "strategy": "single-node", "reason": "only node available" }
```

* `GET /v1/placements?deployment=dpl_1` → list of placements

## Data model

```sql
CREATE TABLE placements (
  id             TEXT PRIMARY KEY,
  deployment_id  TEXT NOT NULL REFERENCES deployments(id),
  replica_index  INT  NOT NULL,
  node_id        TEXT NOT NULL,
  strategy       TEXT NOT NULL,
  reason         TEXT,
  created_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (deployment_id, replica_index)
);
```

## Configuration

```text
FORGE_SCHEDULER_ENABLED     # default true
FORGE_SCHEDULER_STRATEGY    # single-node (only option this step)
FORGE_AUTH_MODE             # dev until epic 09
```

## Security considerations

* Placement API is internal (Control trust boundary); no secrets in requests/decisions
* Unique constraint prevents conflicting placements for the same replica

## Observability

* Log per placement: `{deployment_id, replica_index, node_id, strategy, reason}`
* Metric: `forge_placements_total{strategy=...}`
* Trace span `scheduler.place`

## Failure handling

* No node available → return `409`/typed `NoNodeAvailable` decision (real queueing arrives in 08.04)
* Duplicate placement for the same replica → return existing placement (idempotent)
