## Goal

Assemble `demos/08-multi-node`: run two simulated Runtime agents with fixed slot capacities, deploy four replicas and verify even distribution, then stop one agent and verify its replicas are rescheduled onto the survivor. Wire into `make demo DEMO=08` as the epic acceptance gate.

## Why this step exists

The scheduler pieces (fleet, strategies, affinity, reschedule) only prove the epic when a single command demonstrates distribution and node-loss recovery on the real Control + multi-Runtime stack. This is the gate in `MASTER_PLAN.md`'s demo ownership map.

## Dependencies

* Steps `08.01`–`08.05` — full scheduler with reschedule
* Epic `07-deployment-reconciliation` — reconciler that creates replicas per placement
* Epic `04-forge-runtime` — agents runnable as multiple containers

## Scope

* `demos/08-multi-node/` Compose overlay: Control + two runtime agents (`node-a`, `node-b`) + demo service
* Acceptance script: deploy replicas=4, assert 2+2 distribution; stop `node-b`, assert reschedule to `node-a`
* `make demo DEMO=08` wiring + README

* Reschedule not observed before a deadline → demo fails with diagnostics
* Cleanup trap tears down Compose on exit

## Acceptance criteria

* `make demo DEMO=08` runs both phases and exits 0
* Four replicas distribute evenly (2+2) across two equal-capacity nodes
* Stopping one agent marks it offline and its replicas are rescheduled onto the surviving node (or queued and drained if capacity-bound)
* Assertions use the node fleet and placement APIs (deterministic, not log scraping)
* Compose stack is torn down on exit
